    Modifying Signal Generator

    Overview

    This article gives a brief overview of how to modify the signal generator feature to add custom signal generators to the signal command, how to increase the number of indices in the table, and how to increase the number of channels.

    Note

    For more details on the signal command refer to the Audio Commands page.

    Note

    For signal command usage examples refer to the Signal Generator Example page.

    Details

    Adding Signal Generators

    Follow these steps to add a custom signal generator:

    Step

    Example

    1. In signal_gen.h, add new enum element to SIGNAL_TYPE enum.

    typedef enum SIGNAL_TYPE { SIGNAL_TYPE_NONE, SIGNAL_TYPE_SINE, SIGNAL_TYPE_SAW, SIGNAL_TYPE_TRIANGLE, SIGNAL_TYPE_SQUARE, SIGNAL_TYPE_HEX, SIGNAL_TYPE_IMPULSE, SIGNAL_TYPE_WHITE_NOISE, SIGNAL_TYPE_PINK_NOISE, SIGNAL_TYPE_CUSTOM } SIGNAL_TYPE;

    1. In signal_gen.h, create new structure with the new signal control variables.

    typedef struct { unsigned freq; float amplitude; } SIGNAL_CUSTOM;

    1. In signal_gen.h, add the new structure to the SIGNAL_TABLE union.

    typedef union { SIGNAL_SINE sine; SIGNAL_SAW saw; SIGNAL_TRIANGLE triangle; SIGNAL_SQUARE square; SIGNAL_HEX hex; SIGNAL_IMPULSE impulse; SIGNAL_WHITE white_noise; SIGNAL_PINK pink_noise; SIGNAL_CUSTOM custom; } SIGNAL_TABLE;
    3. In signal_gen.h, add the new structure to the SIGNAL_TABLE union. | typedef union { SIGNAL_SINE sine; SIGNAL_SAW saw; SIGNAL_TRIANGLE triangle; SIGNAL_SQUARE square; SIGNAL_HEX hex; SIGNAL_IMPULSE impulse; SIGNAL_WHITE white_noise; SIGNAL_PINK pink_noise; SIGNAL_CUSTOM custom; } SIGNAL_TABLE; |
    | SYSTEM_AUDIO_TYPE *audioBuf = (SYSTEM_AUDIO_TYPE *) buf; |
    | for(i = 0u; i < samples; i++) { |
    | temp = (i & 1u) ? 1.0f : -1.0f; |
    | temp *= st->pink_noise.amplitude; |
    | audioBuf[i] = (int) (temp * INT_MAX); |
    | } |
    | }`` |

    1. In signal_gen.c, modify getSignal function to call the signal generator function created in step 4.

    case SIGNAL_TYPE_CUSTOM: custom_signal(&sg->sigTbl, buf, samples); break;

    1. In signal_gen.c, modify setSignal function to set the sg->enabled flag when sg->type is equal to the enum value defined in step 1. If needed, initialize the signal generator variables defined in step 2.

    case SIGNAL_TYPE_CUSTOM: sg->enabled = true; break;

    1. In shell_signal in shell_cmd.c, add the new signal to the signal table info switch

    case SIGNAL_TYPE_CUSTOM: printf(" [%d]: CUSTOM, %d Hz, A:%.5f, %s\n", i, sgt->custom.freq, sgt->custom.amplitude, clock_domain_str(domain)); break;

    1. In shell_signal in shell_cmd.c, add a new strcmp case to compare the signal name argument against the desired signal name and set the type variable.

    } else if (strcmp(argv[opt_offset + 2], "custom") == 0) { type = SIGNAL_TYPE_CUSTOM;

    1. (Optional) In shell_signal in shell_cmd.c, add a new case to validate the new signal arguments passed through the signal command. It is suggested to validate the number of arguments and arguments type and range inside this case.

    case SIGNAL_TYPE_CUSTOM: if(argc >= (opt_offset + 4)) { if (argc >= (opt_offset + 3)) { signal.value = atoi(argv[opt_offset + 3]); } if((signal.value == 0) || (signal.value > (SYSTEM_SAMPLE_RATE/2))) { printf("Invalid frequency\n"); return; } } else { printf("Invalid arguments\n"); return; } signal.amplitude = 1.0f; if(argc >= (opt_offset + 5)) { if (argc >= (opt_offset + 4)) { signal.amplitude = atof(argv[opt_offset + 4]); } if((signal.amplitude <= 0) || (signal.amplitude > 1.0f)) { printf("Invalid amplitude\n"); return; } } if(argc > (opt_offset + 5)) { printf("Too many arguments\n"); return; } break;

    1. In shell_signal in shell_cmd.c, inside the semaphore protected section, copy the signal control variables from local variables to the static SIGNAL_GEN structure inside the context structure.

    case SIGNAL_TYPE_CUSTOM: sg[idx].sigTbl.custom.freq = signal.value; sg[idx].sigTbl.custom.amplitude = signal.amplitude; break;

    Increasing Signal Generator Count

    Follow these steps to increase the number of indices in the signal generator table:

    Step

    Example

    1. In signal_gen.h, change the MAX_SIGNAL_GEN macro value.

    #define MAX_SIGNAL_GEN (8)

    1. In route.h, in STREAM_ID enum, add as many new STREAM_ID_SIGNAL[n]_OUT elements as needed.

    typedef enum _STREAM_ID { STREAM_ID_UNKNOWN = 0, STREAM_ID_CODEC_IN, STREAM_ID_CODEC_OUT, STREAM_ID_SPDIF_IN, STREAM_ID_SPDIF_OUT, STREAM_ID_A2B_IN, STREAM_ID_A2B_OUT, STREAM_ID_USB_RX, STREAM_ID_USB_TX, STREAM_ID_WAV_SRC, STREAM_ID_WAV_SINK, STREAM_ID_RTP_RX, STREAM_ID_RTP_TX, STREAM_ID_VBAN_RX, STREAM_ID_VBAN_TX, STREAM_ID_SHARC0_IN, STREAM_ID_SHARC0_OUT, STREAM_ID_SHARC1_IN, STREAM_ID_SHARC1_OUT, STREAM_ID_VU_IN, STREAM_ID_A2B2_IN, STREAM_ID_A2B2_OUT, STREAM_ID_SIGNAL0_OUT, STREAM_ID_SIGNAL1_OUT, STREAM_ID_SIGNAL2_OUT, STREAM_ID_SIGNAL3_OUT, STREAM_ID_SIGNAL4_OUT, STREAM_ID_SIGNAL5_OUT, STREAM_ID_SIGNAL6_OUT, STREAM_ID_SIGNAL7_OUT, STREAM_ID_MAX } STREAM_ID;

    1. In shell_cmds.c, update stream2str to convert all the new enum values to strings.

    case STREAM_ID_SIGNAL0_OUT: str = "SIGNAL0_OUT"; break; case STREAM_ID_SIGNAL1_OUT: str = "SIGNAL1_OUT"; break; case STREAM_ID_SIGNAL2_OUT: str = "SIGNAL2_OUT"; break; case STREAM_ID_SIGNAL3_OUT: str = "SIGNAL3_OUT"; break; case STREAM_ID_SIGNAL4_OUT: str = "SIGNAL4_OUT"; break; case STREAM_ID_SIGNAL5_OUT: str = "SIGNAL5_OUT"; break; case STREAM_ID_SIGNAL6_OUT: str = "SIGNAL6_OUT"; break; case STREAM_ID_SIGNAL7_OUT: str = "SIGNAL7_OUT"; break;

    Increasing Signal Generator Channels

    Follow these steps to increase the number of channels each signal generator will output:

    Step

    Example

    1. In signal_gen.h, create a macro defining the desired number of channels per signal generator.

    <code> #define SIGNAL_GEN_CHANNELS (4) </code>

    1. In signal_audio.c, increase the last array size by the number of supported channels

    <code> static SYSTEM_AUDIO_TYPE signalBuffers[MAX_SIGNAL_GEN][2][SYSTEM_BLOCK_SIZE * SIGNAL_GEN_CHANNELS]; </code>

    1. In signal_audio.c, in xferSignalOutAudio function, assign the new macro to the numChannels output.

    <code> if (!signalSrc->enabled) { *numChannels = 0; *audio = NULL; } else { *audio = signalBuffers[idx][signalSrc->bufferIdx]; *numChannels = SIGNAL_GEN_CHANNELS; } </code>

    1. In signal_gen.c, in sine_wave function, add a loop to write the sample value SIGNAL_GEN_CHANNELS times to the output buffer.

    <code> static void sine_wave(SIGNAL_TABLE *st, void *buf, size_t samples) { float temp; unsigned i, j; SYSTEM_AUDIO_TYPE *audioBuf = (SYSTEM_AUDIO_TYPE *) buf; for(i = 0u; i < samples; i++) { temp = st->sine.amplitude * sinf(st->sine.accum); st->sine.accum += st->sine.phase; if (st->sine.accum >= 2 * M_PI) { st->sine.accum -= 2 * M_PI; } for(j = 0u; j < SIGNAL_GEN_CHANNELS; j++) { audioBuf[i * SIGNAL_GEN_CHANNELS + j] = (int) (temp * INT_MAX); } } } </code>

    1. Repeat the previous step for all signal generator functions
