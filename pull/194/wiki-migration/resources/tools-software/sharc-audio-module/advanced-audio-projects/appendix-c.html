    ====== Appendix C ====== Below describes audio routing in more detail and provides additional details on system configurations, audio configurations, etc. by Audio Starter platform.

    Overview

    The advanced audio projects support several system-level configurations depending on the hardware platform that the software is running on. These differences are usually to account for the differences in physical hardware, but also may vary due to system limitations (or improvements, such as increased memory, better CPU performance, etc.).

    Below you can find more details on the system configuration and relevant hardware mappings for each Audio Starter. Note that unless otherwise specified, these configurations apply only to the latest version of software released.

    Configurations

    ADZS-SC589-MINI

    System Configuration

    Listed below are the overall system settings for the AZDS-SC589-MINI which can be used as a reference when using shell commands, creating A2B projects, etc.

    Configuration

    Value

    Sample Rate

    48kHz

    Block Size

    64

    Max System Channels

    32

    Default A2B Address

    0x68

    Default A2B TWI Port

    TWI0

    Max Routes

    16

    Audio Configuration

    Listed below are the audio specific settings for the AZDS-SC589-MINI. These are tied specifically to each audio source or sink available.

    Source/Sink

    Description

    Source/Sink Offsets

    Audio Settings

    Maximum Audio Channels

    codec

    As a source, this source takes 2 channels of analog audio from LINE IN and places them into TDM slots 0 and 1 to feed into the DSP. As a sink, this sink takes TDM slots 0 and 1 and outputs them to 2 analog output channels on LINE OUT.

    0 = Left 1 = Right

    TDM8 - Clocked Source Bits per channel = 32 Frame sync = Early FS (data MSb delayed 1 Bit-Clock) Assert FS and data on BCLK Falling edge

    IN - 2 OUT - 2

    spdif

    This source takes 2 channels of optical audio and places them into TDM slots 0 and 1 to feed into the DSP.

    0 = Left 1 = Right

    TDM2 - Clocked Source Bits per channel = 32 Frame sync = Early FS (data MSb delayed 1 Bit-Clock) FS 50% duty cycle FS Inverted

    IN - 2 OUT - 2

    a2b

    As a source, this source takes up to 16 channels of upstream A2B audio and places them into TDM slots 0-15 to feed into the DSP. As a sink, this sink takes 0 - 15 TDM slots and feeds them into up to 16 downstream A2B audio channels. The total number of channels for upstream/downstream, up to the maximum, depends on the physical A2B network configuration. See A2B Commands for more details.

    0 = TDM slot 0 1 = TDM slot 1 2 = TDM slot 2 …and so on…

    TDM16 - Clocked Source Bits per channel = 32 Frame sync = Early FS (data MSb delayed 1 Bit-Clock) Assert FS and data on Bit-Clock Falling Edge

    IN - 16 OUT - 16

    usb

    As a source, this source takes up to 16 channels of USB TX Audio from a PC and places it into internal system buffers to be processed at the rate of a clocked source. As a sink, this sink takes up to 16 channels of internal audio buffers that were processed at the rate of a clocked source and outputs them to USB RX to the PC.

    0 = Left Spatial Channel 1 = Right Spatial Channel …and so on…

    NA - Clockless Source Bits per channel = 16

    IN - 16 OUT - 16

    wav

    As a source, this source takes up to 64 channels from a wave file stored in one of the file-systems and places it into internal system buffers to be processed at the rate of a clocked source. As a sink, this sink becomes a recordable wave file to be recorded and stored on a supported file-system. Note that the maximum achievable WAV channels depends on the speed of the SD Card (when used). See SD Test Command for more details.

    0 = Channel 0 1 = Channel 1 …and so on…

    NA - Clockless Source Bits per channel (configurable) = 16 or 32

    IN - 64 OUT - 64

    rtp/vban

    As a source, this source takes up to 32 channels from an Ethernet stream and places it into internal system buffers to be processed at the rate of a clocked source. As a sink, this sink takes audio data from internal system buffers that were processed at the rate of a clocked source and pipes them out to an Ethernet stream.

    0 = Channel 0 1 = Channel 1 …and so on…

    NA - Clockless Source Bits per channel (configurable) = 16 or 32

    IN - 32 OUT - 32

    sharc0/sharc1

    This is a virtual source/sink that allows various audio streams to be routed to one of two SHARC cores for additional processing such as processing for custom algorithms.

    0 = Channel 0 1 = Channel 1 …and so on…

    NA - Clockless Source Bits per channel = 32

    IN - 32 OUT - 32

    vu

    This is a virtual sink (only) that allows various audio streams to be virtualized and viewed in the shell.

    0 = Channel 0 1 = Channel 1 …and so on…

    NA - Clockless Source

    OUT - 64
