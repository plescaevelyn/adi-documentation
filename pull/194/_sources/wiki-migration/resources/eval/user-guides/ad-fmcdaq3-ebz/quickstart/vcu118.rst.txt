AD-FMCDAQ3-EBZ Virtex UltraScale+ VCU118 Quick Start Guide
==========================================================

|VCU118_daq3.jpg| This guide provides some quick instructions (still takes awhile to download, and set things up) on how to setup the AD-FMCDAQ3-EBZ on `VCU118 <https://www.xilinx.com/VCU118>`_

Prerequisites
-------------

Required Hardware
~~~~~~~~~~~~~~~~~

-   `Xilinx VCU118 <https://www.xilinx.com/VCU118>`_ Rev 1.0 or later board
-  :adi:`AD-FMCDAQ3-EBZ <en/design-center/evaluation-hardware-and-software/evaluation-boards-kits/eval-fmcdaq3-ebz.html>` FMC board
-  Ethernet cable
-  2 x Micro-USB cable

Required Software
~~~~~~~~~~~~~~~~~

-  You need a Host PC (Windows or Linux)
-  A UART terminal (Putty/Tera Term/Minicom, etc.), Baud rate 115200 (8N1)
-  Xilinx Vivado 2019.1 or later - Xilinx Software Commandline Tool(XCST)
-  Bitfile and Linux ELF image
-  IIO Scope `Download <https://wiki.analog.com/https/github.com/analogdevicesinc/iio-oscilloscope/releases>`_

.. tip::

   \ :doc:`Pre-build Images for VCU118 and FMCDAQ3. </wiki-migration/resources/tools-software/linux-drivers/platforms/microblaze_loading>`\


Setting up the hardware (VCU118)
--------------------------------

You will need to:

::

   -Get the [[https://www.xilinx.com/VCU118| Xilinx VCU118]]
   {{ resources:eval:user-guides:ad-fmcdaq3-ebz:quickstart:vcu118.png?900 }}
   -Connect the AD-FMCDAQ3-EBZ FMC board to the FPGA carrier **FMC+** socket(J22).
   -Connect USB UART J4 (Micro USB) and USB JTAG J106 (Micro USB) to your host PC.
   -Set the jumpers of SW16 - which controls the JTAG Mode: 1: **off**, 2: **on**, 3: **off**, 4: **on** as seen in the below picture
   {{ resources:eval:user-guides:ad-fmcdaq3-ebz:quickstart:sw16_vcu118.jpg?400 }}
   -Plug the Power Supply into 12V Power input connector (J15).
   -Turn it on.


.. note::

   See `wiki/common <https://wiki.analog.com/wiki/common#esd_warning>`_


Programming the FPGA
--------------------

There are two ways of loading the design. One is using the XCST command line, and the second method is to run the tcl script which takes care of loading the bit file and the Linux image.

2. Loading the design using tcl script
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Run Vivado TCL Shell from Windows start menu → Xilinx Design Suite → Vivado → Vivado TCL Shell. (In Linux, source the settings.sh file first)

::

   ***** Vivado v2021.1 (64-bit)
     *** SW Build 3247384 on Thu Jun 10 19:36:33 MDT 2021
     *** IP Build 3246043 on Fri Jun 11 00:30:35 MDT 2021
       ** Copyright 1986-2021 Xilinx, Inc. All Rights Reserved.

   Vivado% cd \path\to\simpleImage.vcu118_fmcdaq3
   Vivado% xsct run.tcl

   **** Xilinx hw_server v2021.1
     *** Build date : Jun 10 2021 at 20:17:23
       ** Copyright 1986-2021 Xilinx, Inc. All Rights Reserved.
   INFO: hw_server application started
   NFO: To connect to this hw_server instance use url: TCP:127.0.0.1:3121
   Vivado%

.. tip::

   In case of using older releases of VIVADO **xsct run.tcl** command is replaced by **xmd -tcl run.tcl**
