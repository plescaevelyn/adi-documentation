Can't use the 2433MINI on J11 of Carrier Board at the Same Time as the SD Card
------------------------------------------------------------------------------

+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+-----------+
| **Issue Description**                                                                                                                                                                                                                                                                                                  |           |
| Can't discover A2B 2433 Bus (2433MINI) when connected to J11 on the Carrier Board                                                                                                                                                                                                                                      |           |
+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+-----------+
| **Severity** - High                                                                                                                                                                                                                                                                                                    |           |
+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+-----------+
| **Applies to hardware versions**                                                                                                                                                                                                                                                                                       |           |
| *EV-SC598-SOM + EV-SOMCRR-EZKIT*                                                                                                                                                                                                                                                                                       |           |
+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+-----------+
| **Root Cause**                                                                                                                                                                                                                                                                                                         | |image15| |
| The the reset line to the A2B transceiver for the 2433MINI is shared with the SD Card GPIO detect pin on the Carrier board (PB_15). As a result, when the SD Card is inserted, the SD Card GPIO line will be asserted high, resulting in the A2B reset line on the MINI to also be asserted, and thus, put into reset. | |image16| |
+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+-----------+
| **Workaround Steps/Solution**                                                                                                                                                                                                                                                                                          |           |
| If the 2433MINI is required to be used on J11, you cannot use the SD Card (and the SD Card must NOT be inserted).                                                                                                                                                                                                      |           |
| The 2433MINI can be used on J10 but the default I2C addresses need to be adjusted by software.                                                                                                                                                                                                                         |           |
+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+-----------+
| **Planned fix status** - No Plan (Hardware Issue)                                                                                                                                                                                                                                                                      |           |

Error: Failed erasing affected sectors
--------------------------------------

+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+-----------+
| **Issue Description**                                                                                                                                                                                                                         | |image39| |
| When attempting to flash the bootloader(s) on the ADZS-SC589-MINI using the CLDP the device returns *Error: failed erasing affected sectors.*                                                                                                 |           |
+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+-----------+
| **Severity** - Medium                                                                                                                                                                                                                         |           |
+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+-----------+
| **Applies to hardware versions**                                                                                                                                                                                                              |           |
| *ADZS-SC589-MINI v1.5*                                                                                                                                                                                                                        |           |
| *ADZS-SC589-MINI v2.1*                                                                                                                                                                                                                        |           |
+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+-----------+
| **Root Cause**                                                                                                                                                                                                                                |           |
| There is a mismatch between the external flash byte addressing mode between the application/bootloader and the DXE flasher used to re-flash via CLDP, therefore the CLDP command cannot complete the erasing/flash process.                   |           |
+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+-----------+
| **Workaround Steps/Solution**                                                                                                                                                                                                                 | |image40| |
| 1. Remove the JP1 jumper header.                                                                                                                                                                                                              |           |
| 2. Reset the board either by the reset button or by a power cycle.                                                                                                                                                                            |           |
| 3. Re-flash bootloader0 using the instructions :doc:`program-flash </wiki-migration/resources/tools-software/sharc-audio-module/advanced-audio-projects/program-flash>`.                                                                      |           |
| 4. Repeat steps for bootloader1, leaving JP1 disconnected.                                                                                                                                                                                    |           |
| 5. After bootloader flashing is complete, replace JP1 back to positions 1-2 to proceed to application flashing.                                                                                                                               |           |
+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+-----------+
| **Planned fix status** - Fixed                                                                                                                                                                                                                |           |

Application doesn't boot on fresh SAM boards when flashing full LDR via CrossCore Serial Flash Programmer
---------------------------------------------------------------------------------------------------------

+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+-----------+
| **Issue Description**                                                                                                                                                                                                                                                                                                                                                   |           |
| When flashing a fresh SAM board with a full LDR via CrossCore Serial Flash Programmer, application won't boot.                                                                                                                                                                                                                                                          |           |
+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+-----------+
| **Severity** - Blocking                                                                                                                                                                                                                                                                                                                                                 |           |
+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+-----------+
| **Applies to hardware versions**                                                                                                                                                                                                                                                                                                                                        |           |
| ADZS-SC589-MINI - all versions                                                                                                                                                                                                                                                                                                                                          |           |
+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+-----------+
| **Root Cause**                                                                                                                                                                                                                                                                                                                                                          | |image56| |
| The flash chip is required to be in quad mode to boot the application due to the boot setting in the makefile. However, this bit is only set by either the second stage bootloader or the application. A normal bootup starts in the first bootloader and then tries to boot the application, thus there is no path on fresh boards to set the flash chip in quad mode. |           |
+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+-----------+
| **Workaround Steps/Solution**                                                                                                                                                                                                                                                                                                                                           |           |
| Enter the second stage bootloader after a flashing the full LDR on a fresh board by pressing PB1 + PB2 + Reset. After the second stage bootloader is entered, reset the board and the app will boot.                                                                                                                                                                    |           |
+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+-----------+
| **Planned fix status** - TBD                                                                                                                                                                                                                                                                                                                                            |           |

| When SigmaStudio+ is connected to SAM board through USBi interface, application does not properly boot. LED10 blinks twice and then there are no additional LED blinks. |   |
+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------+---+
| **Severity** - Blocking                                                                                                                                                 |   |
+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------+---+
| **Applies to hardware versions**                                                                                                                                        |   |
| ADZS-SC589-MINI - all versions                                                                                                                                          |   |
| EVAL-ADUSB2EBZ (USBi)                                                                                                                                                   |   |
+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------+---+
| **Root Cause**                                                                                                                                                          |   |
| I2C output drivers are open collector and can only assert low. SigmaStudio+ forces USBi I2C output low, thus preventing further communications in the I2C bus.          |   |
+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------+---+
| **Workaround Steps/Solution**                                                                                                                                           |   |
| Disconnect USBi from SAM board.                                                                                                                                         |   |
+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------+---+
| **Planned fix status** - No plan                                                                                                                                        |   |
+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------+---+

+------------------------------------------------------------------------------------------------------------------------+-----------+
| **Severity** - Minor                                                                                                   |           |
+------------------------------------------------------------------------------------------------------------------------+-----------+
| **Applies to hardware versions**                                                                                       |           |
| ALL which support VBAN sink                                                                                            |           |
+------------------------------------------------------------------------------------------------------------------------+-----------+

| Windows Firewall rules may be blocking Voicemeter from receiving the incoming audio stream.                            |           |
+------------------------------------------------------------------------------------------------------------------------+-----------+
| **Workaround Steps/Solution**                                                                                          | |image58| |
| Verify Windows Defender Firewall incoming rules and make sure Voicemeter is enabled. Recommended settings:             |           |
| *Program:* voicemeter.exe                                                                                              |           |
| *Protocol:* UDP                                                                                                        |           |
| *Port:* Any                                                                                                            |           |
+------------------------------------------------------------------------------------------------------------------------+-----------+
| **Planned fix status** - No plan                                                                                       |           |
