Audio Commands
==============

The purpose of the audio shell commands are to allow the users to interact with the audio specific features of the Audio Starter. The sections below describe these commands in more detail.

--------------

Audio Routing (route)
---------------------

+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Description**   | This command allows the user to configure a list/table of audio input to output mappings. It allows the user to pipe an audio input stream, *src*, such as an analog input on the board, to an audio output stream, *dst*, such as an analog output on the board, with a specified number of channels and attenuation.                                                                                                                                                                                                                                                                                                                                                                                                     |
|                   | Audio routing is described in further detail the :doc:`audio routing architecture </wiki-migration/resources/tools-software/sharc-audio-module/advanced-audio-projects/knowledge-base/architecture-audio-routing>` concepts.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| \*\* Syntax \*\*  | *route [<idx> <src> <src offset> <dst> <dst offset> <channels> [attenuation] [mix/set]]* |image4|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Inputs**        | **idx** - This is the index into the table that the user wishes to place audio routing (input to output audio mapping). Note that the lowest index is given the highest priority (i.e. index [00] has a higher priority than index [01] for routes which contain the same destination (see the *mix/set* parameter below).                                                                                                                                                                                                                                                                                                                                                                                                 |
|                   | *Range(integer, required): [0, 15] <- The maximum number of routes configurable may vary depending on the specific Audio Starter project. To determine the number of routes configurable, type route to view the entire table.*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
|                   | //Example:// |image5|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
|                   | **src/dst** - This is the physical audio input/output stream that will be used in the routing table. Streams come from (or pipe to) either a physical hardware source with a physical clock, for example, an ADC/DAC, a SPDIF port, or from/to a clock-less source/sink such as a wave file. Note that not every stream can be used as an input/output (source/sink) and not every stream listed is applicable to the Audio Starter. The range listed is the superset for all Audio Starters. A succinct list can be found by typing *help route* or by checking out our Wiki: :doc:`Audio Feature Matrix </wiki-migration/resources/tools-software/sharc-audio-module/advanced-audio-projects/appendix-a>`.               |
|                   | //Range(string, lower-case, required): [usb, adc, codec, spdif, sharc0, sharc1, wav, rtp, vban, vu, mic, off] //                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
|                   | Example:// *                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
|                   | *\ Note that some sources and sinks are not enabled by default. Some require additional configuration prior to being enabled and routed. See wav, vban and rtp below for more details.//                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|                   | **src/dst offset** - This is the physical starting index of the TDM channel to use for routing a source or a sink (input/output). For example, a value of 0 for *src offset* of the codec source will start mapping from TDM slot 0 and a value of 0 for *sink offset* will map the starting index of the source audio to TDM slot 0 of the sink. Note that additional mapping to physical hardware, for example, left/right channels on a DAC, etc. is dependent on the physical hardware layout. Refer to :doc:`Appendix C </wiki-migration/resources/tools-software/sharc-audio-module/advanced-audio-projects/appendix-c>`                                                                                             |
|                   | //Range(integer, required): [0, N], where N depends on the number of slots configured for a TDM stream (clocked sink/sources), or the number of virtual channels configured for a clock-less source/sink. This is described further in :doc:`Appendix C </wiki-migration/resources/tools-software/sharc-audio-module/advanced-audio-projects/appendix-c>`. //                                                                                                                                                                                                                                                                                                                                                              |
|                   | **channels** - This is the physical number of channels to route. Setting this parameter ultimately specifies the number of channels that are going to be output to the final destination stream.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
|                   | *Range(integer, required): [0, N], where N depends on the destination source's maximum number of channels. See* :doc:`Appendix C </wiki-migration/resources/tools-software/sharc-audio-module/advanced-audio-projects/appendix-c>` *for details.*                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
|                   | **attenuation** - This parameter specifies the amount of dB to attenuate the destination sink *by*. The value passed into this parameter is an attenuation only. This means that the output gain will always be lower/reduced from the input. Even though this parameter accepts both positive and negative values, only the absolute value will be applied.                                                                                                                                                                                                                                                                                                                                                               |
|                   | *Range(integer, optional, default = 0dB): [-120, 120]dB*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|                   | **mix/set** - This parameter allows multiple input sources to to be mixed and routed to the same output. Without this option, if two routes are set with the same destination, the lower priority (idx) route is clobbered.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
|                   | *Range(string, lower-case, optional, default = set): [mix, set]*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Outputs**       | List of routes in the table from indices 0 to Max Routes                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Example Usage** | *route 0 usb 0 codec 0 2 0*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
|                   | This command adds a route at 0th index in audio routing table from USB to codec dst with 0 offsets with 2 channels and 0dB attenuation                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
|                   | |image6|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|                   | *Note that typing **route** without any parameters simply displays the contents of the table and typing **route clear**\ *, clears the contents of the table.*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |

Wave Source/Sink Configuration (wav)
------------------------------------

.. note::

   **Short options support status**:

   
   -  **ADZS-SC589-MINI** - v4.0.0 or greater
   -  **ADZS-SC584-EZLITE** - TBD
   -  **EV-SC594-SOM** - TBD
   -  **EV-SC598-SOM** - TBD
   
   For other variants and versions check if feature is supported in :doc:`Appendix A </wiki-migration/resources/tools-software/sharc-audio-module/advanced-audio-projects/appendix-a>`.


+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Description**   | This command allows the user to configure the wav source or sink file that is going to be used. Wav sources and sinks are not enabled by default as the user must tell the Audio Starter which file to read from or write to, along with the proper settings of the file, number of channels and bits per channel, for example. Users must interact with this command prior to using wav files as a source or a sink in a route.                                                                                                                               |
|                   | WAV Audio Files must be encoded according to `WAV Format <https://en.wikipedia.org/wiki/WAV#:~:text=Waveform%20Audio%20File%20Format%20(WAVE,1991%20by%20IBM%20and%20Microsoft.>`_ and must be encoded in either a 16-bit or 32-bit format. Note that a 24-bit encoded format is NOT supported.                                                                                                                                                                                                                                                                |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| \*\* Syntax \*\*  | *wav [options] <src or sink> <on or off> [file] [channels] [bits]*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|                   | |image13|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| \*\* Inputs \*\*  | **src/sink** - This tells the wav configurator that file stream to be used is either going to be a source (src/input) or the file stream is going to be a sink (dst/output file to be recorded). Only a single source and a single sink can be configured and on at a given time. That is, you cannot have more than one wav source configured and enabled at a time. And you cannot have more than one wav sink configured and enabled at a time. You can, however, have both a wav source streaming and a wav sink recording concurrently.                   |
|                   | *Range(string, lower-case, required): [src, sink]*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|                   | **on/off** - This tells the wav configurator whether the configured wav file will be enabled (on - streaming or recording started) or disabled (off - streaming or recording stopped).                                                                                                                                                                                                                                                                                                                                                                         |
|                   | *Note that any attempt to turn on a wav source/sink when an existing source/sink is already active will be ignored. Sources and sinks that are running must be turned off prior to starting a new stream, even if the stream is different than the original.*                                                                                                                                                                                                                                                                                                  |
|                   | |image14|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
|                   | *Range(string, lower-case, required): [on, off]*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
|                   | **file** - This is the name of the wav file to stream. This file must reside within one of the supported file systems in the system: :doc:`Which file systems are supported? </wiki-migration/resources/tools-software/sharc-audio-module/advanced-audio-projects/appendix-a>`.                                                                                                                                                                                                                                                                                |
|                   | Additionally, for files that reside in the non-default file-system, the drive specifier must be appended to the filename during configuration. If the wav configurator cannot find or open the file, or if the file is not encoded in the proper WAV format, the configuration will be rejected. See the shell command *drive* for more details.                                                                                                                                                                                                               |
|                   | |image15|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
|                   | *Range(string, lower-case, optional, default = src.wav or sink.wav): <filename>.wav, where filename can be any string.*                                                                                                                                                                                                                                                                                                                                                                                                                                        |
|                   | **channels** - (Not a valid input for *src*. Only valid for *sink*). This is the number of channels to record to the wav file.                                                                                                                                                                                                                                                                                                                                                                                                                                 |
|                   | *Range(integer, optional, default = 2): [1, Maximum Audio Channels], see* :doc:`Audio Configuration </wiki-migration/resources/tools-software/sharc-audio-module/advanced-audio-projects/appendix-c>` *for details.*                                                                                                                                                                                                                                                                                                                                           |
|                   | **bits** - (Not a valid input for *src*. Only valid for *sink*). This is the number of bits per channel to record to the wav file.                                                                                                                                                                                                                                                                                                                                                                                                                             |
|                   | *Range(integer, optional, default = 16): [16 or 32], see* :doc:`Audio Configuration </wiki-migration/resources/tools-software/sharc-audio-module/advanced-audio-projects/appendix-c>` *for details.*                                                                                                                                                                                                                                                                                                                                                           |
|                   | **OPTIONS**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
|                   | **-b <bits>**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
|                   | This optional argument has the same behavior as the **bits** positional argument. If both optional and positional arguments are used, optional argument will be ignored.                                                                                                                                                                                                                                                                                                                                                                                       |
|                   | **-c <channels>**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
|                   | This optional argument has the same behavior as the **channels** positional argument. If both optional and positional arguments are used, optional argument will be ignored.                                                                                                                                                                                                                                                                                                                                                                                   |
|                   | **-d <domain>**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
|                   | This option will set the clock domain of the system that the source or sink will use.                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
|                   | *Range(string, lower-case, optional, default = “system”): [a2b, system]*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|                   | **-f <file>**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
|                   | This optional argument has the same behavior as the **file** positional argument. If both optional and positional arguments are used, optional argument will be ignored.                                                                                                                                                                                                                                                                                                                                                                                       |
|                   | **-i**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
|                   | This option will enable the file name count auto-increment for files created by the *wav sink on* command. Option only needs to be given once along with the file name to the *wav* command. Subsequent calls to the *wav sink on* without the file option will increase the file count in the file name. If the auto-increment option is not enabled, the sink file will be overwritten each time the *wav sink on* command is called.                                                                                                                        |
|                   | **-l <loop count>**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
|                   | This option can be used to reproduce a source **file** the determined number of times. If not set, source file will be reproduced indefinitely. Option will be ignored for sink files.                                                                                                                                                                                                                                                                                                                                                                         |
|                   | *Range(integer, optional, default = infinite): [0, 2147483647]*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Outputs**       | Source/Sink Configuration Status:                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
|                   | *src/sink: ON/OFF, <filename.wav>, #bits per channel, #channels, clock domain*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
|                   | |image16| |image17|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
|                   | For sources, #channels is the number of channels encoded in the wav file and #bits is the number of bits encoded in the wav file. These parameters cannot be overwritten.                                                                                                                                                                                                                                                                                                                                                                                      |
|                   | For sinks, the same parameters are the values that were previously configured from the inputs section.                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Example Usage** | *wav src on 1kHz_USB.wav*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
|                   | This turns on wav source with the filename **1kHz_USB.wav** located in the default file system.                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
|                   | *wav sink on fileRecord.wav*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|                   | This creates a new file in the default file system called **fileRecord.wav** with audio parameters 16-bits per channel and 2 channels of audio and starts recording into the wav file.                                                                                                                                                                                                                                                                                                                                                                         |
|                   | |image18|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
|                   | *Note that for audio sinks, to access the recorded wav file, the sink MUST be turned off prior to access. Any attempt to access the recorded wav file prior to turning it off will fail.*                                                                                                                                                                                                                                                                                                                                                                      |
|                   | *wav -f source.wav -l 5 src on*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
|                   | This will play *source.wav* 5 times.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
|                   | *wav -f sink.wav -c 4 -b 32 sink on*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
|                   | This will store a 4 channel 32-bit stream to *sink.wav*.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|                   | *wav -i -f file.wav sink on*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|                   | *wav sink off*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
|                   | *wav sink on*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
|                   | *wav sink off*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
|                   | *wav sink on*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
|                   | *wav sink off*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
|                   | This will generate thre files: file.wav, file_1.wav, file_2.wav.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |

VBAN Source/Sink Configuration (vban)
-------------------------------------

+-------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Description**   | This command allows the user to configure the VBAN RX or TX stream that is going to be used. VBAN streams are not enabled by default as the user must tell the Audio Starter which Ethernet port to pipe to or from, along with the proper settings of the stream, number of channels and bits per channel, for example. Users must interact with this command prior to using VBAN as a source or a sink in a route.                                                              |
|                   | VBAN streams must conform to the `VBAN protocol <https://vb-audio.com/Voicemeeter/VBANProtocol_Specifications.pdf>`_ and must be encoded in either a 16-bit or 32-bit format. Note that a 24-bit encoded format is NOT supported.                                                                                                                                                                                                                                                 |
+-------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| \*\* Syntax \*\*  | *vban <rx or tx> <on or off> <ip> [port] [channels] [bits]*                                                                                                                                                                                                                                                                                                                                                                                                                       |
|                   | |image22|                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
+-------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| \*\* Inputs \*\*  | **rx/tx** - This tells the vban configurator that stream to be used is either going to be a source (Rx/input) or the stream is going to be a sink (Tx/output). Only a single RX and a single TX can be configured and on at a given time. That is, you cannot have more than one RX configured and enabled at a time. And you cannot have more than one TX configured and enabled at a time. You can, however, have both an RX and a TX streaming concurrently.                   |
|                   | *Range(string, lower-case, required): [rx, tx]*                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|                   | **on/off** - This tells the vban configurator whether the configured stream will be enabled (on) or disabled (off).                                                                                                                                                                                                                                                                                                                                                               |
|                   | *Range(string, lower-case, required): [on, off]*                                                                                                                                                                                                                                                                                                                                                                                                                                  |
|                   | **ip** - This is the IP address to stream to (Audio Starter TX to Ethernet Port) or the IP Address to stream from (Ethernet Port to Audio Starter RX). For streaming TX to the Ethernet port, the IP address must be in the same domain of the Audio Starter. :doc:`How do I get these IP addresses? </wiki-migration/resources/tools-software/sharc-audio-module/advanced-audio-projects/examples-ethernet-audio>`                                                               |
|                   | *Range(string, lower-case, required): <IPv4 Standard Address>*                                                                                                                                                                                                                                                                                                                                                                                                                    |
|                   | **port** - This is the unsigned, 16-bit port value to be used with the IP address.                                                                                                                                                                                                                                                                                                                                                                                                |
|                   | *Range(integer, optional, default = 6980): [0, 65535]*                                                                                                                                                                                                                                                                                                                                                                                                                            |
|                   | **channels** - This is the number of VBAN channels to stream.                                                                                                                                                                                                                                                                                                                                                                                                                     |
|                   | *Range(integer, optional, default = 2): [1, Maximum Audio Channels], see* :doc:`Audio Configuration </wiki-migration/resources/tools-software/sharc-audio-module/advanced-audio-projects/appendix-c>` *for details.*                                                                                                                                                                                                                                                              |
|                   | **bits** - This is the number of bits per channel to stream.                                                                                                                                                                                                                                                                                                                                                                                                                      |
|                   | *Range(integer, optional, default = 16): [16 or 32]*                                                                                                                                                                                                                                                                                                                                                                                                                              |
+-------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Outputs**       | RX/TX Configuration Status:                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|                   | *Rx/Tx: ON/OFF, <IPv4 Address>:Port, #bits per channel, #channels, clock domain*                                                                                                                                                                                                                                                                                                                                                                                                  |
|                   | |image23|                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
+-------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Example Usage** | *vban rx on 192.168.2.2 6980 2 16* and *vban rx domain system*                                                                                                                                                                                                                                                                                                                                                                                                                    |
|                   | This turns on a VBAN RX stream with IP Address 192.168.2.2, with port number 6980, 2 channels and 16-bit audio.                                                                                                                                                                                                                                                                                                                                                                   |
|                   | |image24|                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
|                   | *Refer to* :doc:`Ethernet Audio Examples </wiki-migration/resources/tools-software/sharc-audio-module/advanced-audio-projects/examples-ethernet-audio>` *for more examples of VBAN usage!*                                                                                                                                                                                                                                                                                        |

Volume Unit Meters Display (vu) - DEPRECATED
--------------------------------------------

+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Description**   | This command displays a relative audio signal level for any source audio that it is routed to. This command MUST be used in conjunction with a route command for a signal level to be present. Although this command cannot show audio distortion or clipping necessarily, it is a very useful command for being able to visualize audio on up to 64 channels, where audio may be limited otherwise due to hardware availability. For example, a codec may only support 2 physical channels, but a source with more than 2 channels may be visualized using the VU Meters. |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| \*\* Syntax \*\*  | vu                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
|                   | |image35|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Inputs**        | \**Option #1**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|                   | **No Input** - Will display 32 channels of VU Meters                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|                   | \**Option #2**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |

Signal generator (signal)
-------------------------

.. note::

   **Support status**:

   
   -  **ADZS-SC589-MINI** - TBD
   -  **ADZS-SC584-EZLITE** - TBD
   -  **EV-SC594-SOM** - TBD
   -  **EV-SC598-SOM** - TBD
   


+-------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Description**   | The purpose of this shell command is to use the SAM software capabilities to generate some of the most-commonly-used audio signals as sources that can be routed to any sink.                                         |
|                   | Signal generator can generate four different one-channel signals at the same time. Signals can be routed independently or mixed together via the route command.                                                       |
|                   | Available signals:                                                                                                                                                                                                    |
|                   | \* Sine                                                                                                                                                                                                               |
|                   | \* Saw                                                                                                                                                                                                                |
|                   | \* Triangle                                                                                                                                                                                                           |
|                   | \* Square                                                                                                                                                                                                             |
|                   | \* White noise                                                                                                                                                                                                        |
|                   | \* Pink noise                                                                                                                                                                                                         |
|                   | \* Constant (hex) value                                                                                                                                                                                               |
|                   | \* Impulse response                                                                                                                                                                                                   |
+-------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| \*\* Syntax \*\*  | // signal [ [options] <idx> <type> [signal specific] ]//                                                                                                                                                              |
|                   | // signal [options] <idx> sine <frequency> [amplitude]//                                                                                                                                                              |
|                   | // signal [options] <idx> saw <frequency> [amplitude]//                                                                                                                                                               |
|                   | // signal [options] <idx> triangle <frequency> [amplitude]//                                                                                                                                                          |
|                   | // signal [options] <idx> square <frequency> [amplitude]//                                                                                                                                                            |
|                   | // signal [options] <idx> hex <value>//                                                                                                                                                                               |
|                   | // signal [options] <idx> imp [impulse] [period]//                                                                                                                                                                    |
|                   | // signal [options] <idx> white [amplitude]//                                                                                                                                                                         |
|                   | // signal [options] <idx> pink [amplitude]//                                                                                                                                                                          |
|                   | |image65|                                                                                                                                                                                                             |
+-------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Inputs**        | **GENERAL**                                                                                                                                                                                                           |
|                   | **idx** - This selects the signal generator index                                                                                                                                                                     |
|                   | *Range(integer, required): [0, 3].*                                                                                                                                                                                   |
|                   | **type** - This selects the signal type to be generated by the signal generator.                                                                                                                                      |
|                   | //Range(string, required): [sine, saw, triangle, square, hex, imp, white, pink, off] //                                                                                                                               |
|                   | **SIGNAL SPECIFIC**                                                                                                                                                                                                   |
|                   | **frequency** - This sets the signal frequency in Hertz.                                                                                                                                                              |
|                   | *Range(integer, required): [0, 24000]*                                                                                                                                                                                |
|                   | **amplitude** - This sets the signal amplitude.                                                                                                                                                                       |
|                   | // Range(float, optional, default = 1.0): [0.0, 1.0] //                                                                                                                                                               |
|                   | **value** - This sets the constant hex value to be output by the signal generator.                                                                                                                                    |
|                   | // Range(hex, 32-bits, required): [0x1, 0xFFFFFFFF] //                                                                                                                                                                |
|                   | **impulse** - This sets the impulse response value to be output by the signal generator.                                                                                                                              |
|                   | // Range(float, optional, default = 1.0): [-1.0, 1.0]//                                                                                                                                                               |
|                   | **period** - This sets the impulse response periodicity in miliseconds. If no value is given, impulse response will only be output once.                                                                              |
|                   | //Range(integer, optional, default = 0): [1, 2147483647] //                                                                                                                                                           |
|                   | **OPTIONS**                                                                                                                                                                                                           |
|                   | **-d <domain>**                                                                                                                                                                                                       |
|                   | This option will set the clock domain of the signal generator. This option can be set individually for each generator index.                                                                                          |
|                   | *Range(string, lower-case, optional, default = "system"): [a2b, system]*                                                                                                                                              |
+-------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Outputs**       | List of generated signals from indices 0 to 3.                                                                                                                                                                        |
+-------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Example Usage** | // signal //                                                                                                                                                                                                          |
|                   | Displays the signal generator information table.                                                                                                                                                                      |
|                   | |image66|                                                                                                                                                                                                             |
|                   | // signal clear//                                                                                                                                                                                                     |
|                   | Clears all entries in the signal generator table (i.e. turns off all signal generators).                                                                                                                              |
|                   | See :doc:`Signal Generator Example </wiki-migration/resources/tools-software/sharc-audio-module/advanced-audio-projects/examples-signal-generator>` for a more thorough example on how to generate and route signals. |

Asynchrnous Sample Rate Converter (asrc)
----------------------------------------

.. note::

   **Support status**:

   
   -  **ADZS-SC589-MINI** - TBD
   -  **ADZS-SC584-EZLITE** - TBD
   -  **EV-SC594-SOM** - TBD
   -  **EV-SC598-SOM** - TBD
   


+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Description**   | This command allows the user to configure the Asynchronous Sample Rate Converter in the SAM. Users may then route other audio signal to the ASRC sink and from the ASRC source. ASRC may be used to indirectly route a sink to a source working on different clock domains (e.g. A2B to CODEC). |
|                   | ASRC command supports two different conversion methods:                                                                                                                                                                                                                                         |
|                   | - Add/drop samples                                                                                                                                                                                                                                                                              |
|                   | - High quality (ADI propietary library required)                                                                                                                                                                                                                                                |
+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| \*\* Syntax \*\*  | // asrc <0                                                                                                                                                                                                                                                                                      |
+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Inputs**        | **0/1** - This selects the ASRC instance to be configured.                                                                                                                                                                                                                                      |
|                   | *Range(integer, required): [0, 1]*                                                                                                                                                                                                                                                              |
|                   | **on/off** - This enables/disables the specified ASRC instance.                                                                                                                                                                                                                                 |
|                   | *Range(string, lower-case, required): [on, off]*                                                                                                                                                                                                                                                |
|                   | **type** - This selects the conversion method to be used by the ASRC instance.                                                                                                                                                                                                                  |
|                   | *Range(string, lower-case, optional, default = ad): [ad, hq]*                                                                                                                                                                                                                                   |
|                   | **cdin** - This sets the input clock domain.                                                                                                                                                                                                                                                    |
|                   | *Range(string, lower-case, optional, default = system): [system, a2b, rtp, vban, tdm]*                                                                                                                                                                                                          |
|                   | **cdout** - This sets the output clock domain.                                                                                                                                                                                                                                                  |
|                   | *Range(string, lower-case, optional, default = system): [system, a2b, rtp, vban, tdm]*                                                                                                                                                                                                          |
|                   | **channels** - This sets the number of channels to be converted by the ASRC. This applies to both input and output.                                                                                                                                                                             |
|                   | *Range(integer, optional, default = 2): [0, 32]*                                                                                                                                                                                                                                                |
|                   | **quality** - This sets the quality level of the converter. It does not have any impact on the Add/Drop or High Quality converters but may be used with other custom converters.                                                                                                                |
|                   | *Range(integer, optional, default = 10): [0, 10]*                                                                                                                                                                                                                                               |
|                   | **rbsize** - This sets the number of samples to be held in the ring buffer. Setting this value to 0 will automatically set the size of the ring buffer.                                                                                                                                         |
|                   | *Range(integer, optional, default = 0): [256, ]*                                                                                                                                                                                                                                                |
|                   | **mbsize** - This sets the maximum incoming block size. Setting this value to 0 will automatically set the block size to be equal to the clock domain block size.                                                                                                                               |
|                   | *Range(integer, optional, default = 0): [64, ]*                                                                                                                                                                                                                                                 |
+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Outputs**       | ASRC Settings                                                                                                                                                                                                                                                                                   |
|                   | |image68|                                                                                                                                                                                                                                                                                       |
+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Example Usage** | // asrc //                                                                                                                                                                                                                                                                                      |
|                   | Displays ASRC settings                                                                                                                                                                                                                                                                          |
|                   | // asrc 0 on ad a2b system 2 10 0 0 //                                                                                                                                                                                                                                                          |
|                   | This enables the first instance of the ASRC to convert two channels coming from the A2B clock domain into the System clock domain while auto-sizing the buffer sizes.                                                                                                                           |
