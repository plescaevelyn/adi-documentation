Filesystem Commands
===================

The commands listed below provide a `UNIX-like <https://en.wikipedia.org/wiki/List_of_POSIX_commands>`_ set of commands that allow the user to interact with the :doc:`file systems </wiki-migration/resources/tools-software/sharc-audio-module/advanced-audio-projects/appendix-a>` of the Audio Starter. The file systems are used for storing various files that support the rest of the Audio Starter features, such as wav files, A2B commandlist files, etc.

--------------

Device Drive Specifier (drive)
------------------------------

+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Description**   | The purpose of the this command is to be able to display the device drive specifiers, the default file system and allow the user to update the default file system, where applicable. This is useful as the majority of the Audio Starter projects support more than one file system. As a result, files used within the Audio Starter may potentially reside in a file-system that is different than the default. In these cases, access to these files requires the addition of a device drive specifier, such as *sd:* or *sf:* to indicate which file-system is being accessed. |
|                   | For example, on an Audio Starter which supports an SD Card (physical removable hardware) and SPIFFS (external flash storage, not removable, connected to the processor), the SD Card may be the default file system. In this case, accessing a file on the SD Card only requires the filename (though appending the SD Card specifier to the filename doesn't hurt anything).                                                                                                                                                                                                       |
|                   | |image8|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
|                   | |image9|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
|                   | However, accessing a file that resides on the non-default file system requires the device drive specifier to be appended. This is also true for many of the subsequent commands below as well.                                                                                                                                                                                                                                                                                                                                                                                      |
|                   | |image10|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
|                   | This command *drive* provides the details of the device drive specifiers:                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
|                   | |image11|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
|                   | //Note that it is possible for a file-system that is supported to not show up in this list for a few reasons:                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|                   | 1) The device may not be mounted - This could occur if the device is disconnected, in the case of an SD Card.                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|                   | 2) The device is corrupted and needs to be reformatted. //                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
|                   | The command *syslog* can provide more details about removable storage devices:                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
|                   | |image12|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
|                   | and the command *fsck* can provide more details about the internal flash file system.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| \*\* Syntax \*\*  | *drive <device> <action>*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
|                   | |image13|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Inputs**        | **device** - This is the device drive identifier that a valid action will be taken on. For a list of device drive identifiers, type *drive* with no inputs.                                                                                                                                                                                                                                                                                                                                                                                                                         |
|                   | *Range(string, lower-case, required): <drive>, where the available list comes from the output of **drive** with no inputs>*                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
|                   | **action** - The is the type of action to take on the specified drive.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
|                   | *Range(string, lower-case, required): [default]*.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Outputs**       | A list of available device drive identifiers.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Example Usage** | *drive*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|                   | This command will list the available device drive identifiers.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
|                   | *drive sf: default*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
|                   | This command will set the default file-system to *sf:* (SPIFFS) instead of *sd:* (SD Card).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
|                   | |image14|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |

Directory List (ls)
-------------------

+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Description**   | This command lists the contents of one of the available file-systems. This command, when no inputs are provided, will list the contents of the default file-system, or when a file-system is provided, will list the contents of that file system. |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| \*\* Syntax \*\*  | *ls*                                                                                                                                                                                                                                               |
|                   | |image18|                                                                                                                                                                                                                                          |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Inputs**        | **device** - This is the device drive identifier that will be used to display the contents of the directory.                                                                                                                                       |
|                   | *Range(string, lower-case, optional, default = <default file-system>): <device>, where device is one of the outputs from the **drive** command.*                                                                                                   |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Outputs**       | A list of files in the given file-system.                                                                                                                                                                                                          |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Example Usage** | *ls*                                                                                                                                                                                                                                               |
|                   | |image19|                                                                                                                                                                                                                                          |
|                   | *ls sf:*                                                                                                                                                                                                                                           |
|                   | |image20|                                                                                                                                                                                                                                          |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

Concatenate (cat)
-----------------

+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Description**   | This command prints the contents of a file to the standard output. To print a file that resides in the non-default file-system, append the device drive identifier to the filename. See the command *drive* for more details. |
+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| \*\* Syntax \*\*  | *cat <filename1> [<filename2> ...]*                                                                                                                                                                                           |
|                   | |image24|                                                                                                                                                                                                                     |
|                   | |image25|                                                                                                                                                                                                                     |
+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Inputs**        | **filename1** - This is the file whose contents are to be printed to standard out. Note that files of binary type or non-text type files should not be used.                                                                  |
|                   | *Range(string, required): any*                                                                                                                                                                                                |
+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Outputs**       | The contents of the file                                                                                                                                                                                                      |
+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Example Usage** | *cat pushbtn1.cmd pushbtn2.cmd*                                                                                                                                                                                               |
|                   | Will display the contents of pushbtn1.cmd and pushbtn2.cmd                                                                                                                                                                    |
|                   | and                                                                                                                                                                                                                           |
|                   | *cat pushbtn1.cmd pushbtn2.cmd pushbtn1.cmd*                                                                                                                                                                                  |
|                   | Will display the contents of pushbtn1.cmd and pushbtn2.cmd and pushbtn1.cmd again                                                                                                                                             |
|                   | |image26|                                                                                                                                                                                                                     |
+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

File Compare (cmp)
------------------

+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Description**   | This command compares two files, stopping at the first mismatched byte. To compare files that resides in the non-default file-system, append the device drive identifier to the filename. See the command *drive* for more details. |
+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| \*\* Syntax \*\*  | *cmp <filename1> <filename2>*                                                                                                                                                                                                       |
+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Inputs**        | **filename1** - The first file to compare                                                                                                                                                                                           |
|                   | *Range(string, required): any*                                                                                                                                                                                                      |
|                   | **filename2** - The second file to compare                                                                                                                                                                                          |
|                   | *Range(string, required): any*                                                                                                                                                                                                      |
+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Outputs**       | Displays the comparison result, reporting the first mismatched byte.                                                                                                                                                                |
+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Example Usage** | *cmp 1kHz_USB.wav pushbtn1.cmd*                                                                                                                                                                                                     |
|                   | This will compare two different files, in the same default file-system.                                                                                                                                                             |
|                   | |image29|                                                                                                                                                                                                                           |
|                   | *cmp 1kHz_USB.wav sf:1kHz.wav*                                                                                                                                                                                                      |
|                   | This will compare two equal files, each residing in different file-systems.                                                                                                                                                         |
|                   | |image30|                                                                                                                                                                                                                           |
+-------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

File-System Integrity Check (fsck)
----------------------------------

+-------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Description**   | This command checks the integrity of the internal file-system. This is useful for checking the file-system for corruption or checking to see if the file-system is formatted in a way that makes it ready to use files. The internal file-system may be corrupted or not ready to be used as a file-system if it was previously loaded with other data, for another project, for example, using the same hardware. |
+-------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| \*\* Syntax \*\*  | *fsck <device>*                                                                                                                                                                                                                                                                                                                                                                                                    |
|                   | |image33|                                                                                                                                                                                                                                                                                                                                                                                                          |
+-------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Inputs**        | **device** - This is the device drive identifier of the file-system to be formatted. The range available is listed directly in the usage. Note that removeable file-systems like an SD Card are NOT supported by this command.                                                                                                                                                                                     |
|                   | *Range(string, lower-case, required): Listed directly in usage*                                                                                                                                                                                                                                                                                                                                                    |
+-------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Outputs**       | The status of the integrity check.                                                                                                                                                                                                                                                                                                                                                                                 |
+-------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Example Usage** | *fsck sf:*                                                                                                                                                                                                                                                                                                                                                                                                         |
|                   | |image34|                                                                                                                                                                                                                                                                                                                                                                                                          |
|                   | *Note that if the status reports as corrupted, the file-system can be formatted via the **format** command below.*                                                                                                                                                                                                                                                                                                 |
+-------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

Disk Formatting (format)
------------------------

+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Description**   | This command erases all of the contents of the given file-system and creates a new file system to be used. This can be useful if the file-system is corrupted or if the user wants to quickly delete everything from the file-system without having to manually remove each individual file. |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| \*\* Syntax \*\*  | *format <device>*                                                                                                                                                                                                                                                                            |
|                   | |image37|                                                                                                                                                                                                                                                                                    |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Inputs**        | **device** - This is the device drive identifier of the file-system to be formatted. The range available is listed directly in the usage. Note that removeable file-systems like an SD Card are NOT supported by this command.                                                               |
|                   | *Range(string, lower-case, required): Listed directly in usage*                                                                                                                                                                                                                              |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Outputs**       | None                                                                                                                                                                                                                                                                                         |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Example Usage** | *format sf:*                                                                                                                                                                                                                                                                                 |
|                   | |image38|                                                                                                                                                                                                                                                                                    |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

Disk Free Space (df)
--------------------

+-------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Description**   | This command lists all of the available file-systems, the size of the file-system, how much space has been used, in bytes, how much space is left, in bytes and the percentage used. |
+-------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| \*\* Syntax \*\*  | *df [device]*                                                                                                                                                                        |
|                   | |image41|                                                                                                                                                                            |
+-------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Inputs**        | **device** - This is the device drive identifier of the file-system whose disk space statistics will be displayed. The range available is listed directly in the usage.              |
|                   | *Range(string, lower-case, optional): Listed directly in usage*                                                                                                                      |
+-------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Outputs**       | Shows total size, used and available size of the device                                                                                                                              |
+-------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Example Usage** | *df*                                                                                                                                                                                 |
|                   | Displays all of the statistics for all available devices                                                                                                                             |
|                   | *df sf:*                                                                                                                                                                             |
|                   | Displays statistics for just *sf:*                                                                                                                                                   |
|                   | *df sd:*                                                                                                                                                                             |
|                   | Displays just statistics for *sd:*                                                                                                                                                   |
|                   | |image42|                                                                                                                                                                            |

File Remove (rm)
----------------

+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Description**   | This command removes a file or files from the file-system. To remove files that reside in the non-default file-system, append the device drive identifier to the filename. See the command *drive* for more details. |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| \*\* Syntax \*\*  | //rm <file1> <file2> ... //                                                                                                                                                                                          |
|                   | |image45|                                                                                                                                                                                                            |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Inputs**        | **file1** - The first file to remove                                                                                                                                                                                 |
|                   | *Range(string, required): any*                                                                                                                                                                                       |
|                   | *Note that subsequent files are optional.*                                                                                                                                                                           |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Outputs**       | None                                                                                                                                                                                                                 |
+-------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Example Usage** | *rm sf:1kHz.wav*                                                                                                                                                                                                     |
|                   | This command removes the file *1kHz.wav* from the non-default file-system *sf:*.                                                                                                                                     |
|                   | |image46|                                                                                                                                                                                                            |

File Copy (cp)
--------------

+-------------------+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Description**   | This command copies one file (source) in a file-system to another file (destination) in a file-system. To copy files to or from the non-default file-system, append the device drive identifier to the filename. See the command *drive* for more details. |
+-------------------+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| \*\* Syntax \*\*  | *cp <src> <dst>*                                                                                                                                                                                                                                           |
|                   | |image49|                                                                                                                                                                                                                                                  |
+-------------------+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Inputs**        | **src** - This is the file which will be copied.                                                                                                                                                                                                           |
|                   | *Range(string, required): any*                                                                                                                                                                                                                             |
|                   | **dst** - This is the name (and appended file-system, where applicable) of the copied file. **Caution** - If a file of the same name already exists, it will be overwritten without warning!.                                                              |
|                   | *Range(string, required): any*                                                                                                                                                                                                                             |
+-------------------+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Outputs**       | None                                                                                                                                                                                                                                                       |
+-------------------+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Example Usage** | *cp 1kHz_USB.wav sf:1kHz.wav*                                                                                                                                                                                                                              |
|                   | This command will copy the file *1kHz_USB.wav* from the default file-system and place it into the *sf:* file-system with the name *1kHz.wav*.                                                                                                              |
|                   | |image50|                                                                                                                                                                                                                                                  |

SD Card Speed Test (sdtest)
---------------------------

+-------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Description**   | The purpose of this command is to test the read/write speed of the SD Card. This is useful for verifying if the current SD Card speeds are sufficient for the throughput required for the type of wav file that you intend to play. For example, wav files with a higher channel count or a higher bit-rate, may require faster throughput to continue playing without distortion or dropping samples.                                                                                  |
|                   | Typically the minimum SD Card recommended is a `Class 10 SD Card <https://www.kingston.com/en/blog/personal-storage/memory-card-speed-classes>`_, which is supposed to be able to achieve 10MB/s, some cards throughput can slow down with age, fullness and formatting. Therefore, when running wav files at a higher bit-rate or higher channel count, it is recommended to verify that the SD Card throughput is going to be able to meet the speeds necessary to stream the audio.  |
|                   | By default, the test will check the bandwidth for 5 seconds and spit out the write and read bandwidth in *KB/s (Kilo-Bytes per second)*                                                                                                                                                                                                                                                                                                                                                 |
|                   | The throughput requirement for a wav file can be found usually one of two ways:                                                                                                                                                                                                                                                                                                                                                                                                         |
|                   | 1) On Windows, you can find the *Audio Bit rate* by right-clicking on the wav file and selecting the *Details* view.                                                                                                                                                                                                                                                                                                                                                                    |
|                   | |image79|                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
|                   | Note that this parameter may not be in the same units as the SD Test. In this case, the parameter is in kbps (kilo-bits per second). To convert to Bytes to match the units of the SD Test, divide this parameter by 8 (8 bits in a byte).                                                                                                                                                                                                                                              |
|                   | Or 2) If you know the number of channels, sampling rate and number of bits per channel, you can multiple these three numbers together to get the bit-rate:                                                                                                                                                                                                                                                                                                                              |
|                   | *16 channels \* 48kHz \* 16-bits per channel = 12288kbps*. Again, divide this number by 8 to get the throughput in KB/s.                                                                                                                                                                                                                                                                                                                                                                |
|                   | The calculated bit-rate of the wav file should be less than the read result of the SD Test.                                                                                                                                                                                                                                                                                                                                                                                             |
+-------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| \*\* Syntax \*\*  | *sdtest [ms]*                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
|                   | |image80|                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
+-------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Inputs**        | **ms** - This is the number of milliseconds to run the throughput test for.                                                                                                                                                                                                                                                                                                                                                                                                             |
|                   | *Range(integer, optional, default = 5000ms): [0, MAX_U32]*                                                                                                                                                                                                                                                                                                                                                                                                                              |
+-------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Outputs**       | Read/Write throughput results in KB/s                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
+-------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Example Usage** | *sdtest*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
|                   | |image81|                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
|                   | *sdtest 1000*                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
|                   | |image82|                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
+-------------------+-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

Date and Time of Day (date)
---------------------------

+-------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Description**   | This shell command either displays the date and time of day, or accepts a new date and time of day. The format for date and time of day follows the `RFC3339 Date/Time String Format <https://medium.easyread.co/understanding-about-rfc-3339-for-datetime-formatting-in-software-engineering-940aa5d5f68a>`_ and by default at initial hardware power on, starts counting from the `FAT epoch <https://en.wikipedia.org/wiki/File_Allocation_Table>`_ which is 1-Jan-1980, which is 10 years + 2 days after the UNIX epoch of 1-Jan-1970. The 1980 epoch keeps FatFS happy.   |
|                   | For more details about the formatting, refer to https://www.timeanddate.com/date/timeduration.html                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
+-------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| \*\* Syntax \*\*  | *date <now>*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|                   | |image86|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
+-------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Inputs**        | **now** - This is the RFC3339 string required for setting a new date and time.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
|                   | *Range(string, required): <RFC3339 String>*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
+-------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Outputs**       | The date and time in RFC3339 Format                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
+-------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| **Example Usage** | *date*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
|                   | |image87|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
|                   | *This displays the date of 1-Jan-1980 at time 00:11:14 and timezone of UTC+0.*                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
|                   | // date 1996-12-19T16:39:57//                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
|                   | |image88|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
|                   | *This sets the date, time and timezone to 39 minutes and 57 seconds after the 16th hour of December 19th, 1996 and timezone of UTC+0.*                                                                                                                                                                                                                                                                                                                                                                                                                                         |
