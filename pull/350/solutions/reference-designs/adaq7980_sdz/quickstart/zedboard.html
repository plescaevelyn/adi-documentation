ZedBoard Quick Start Guide

This guide provides instructions for setting up the EVAL-ADAQ7980-SDZ with the ZedBoard development platform using no-OS software.

Overview

The ZedBoard is a Xilinx Zynq-7000 SoC development board featuring:

  • Zynq-7000 XC7Z020-CLG484-1 SoC

  • Dual ARM Cortex-A9 processors

  • Artix-7 FPGA fabric

  • FMC LPC connector for expansion boards

  • USB UART and JTAG interfaces

  • Ethernet, HDMI, and various peripherals

The EVAL-ADAQ7980-SDZ connects to the FMC connector and uses the SPI Engine framework for high-performance data acquisition.

Hardware Setup

Before building the project, set up the hardware:

  1. Verify Prerequisites are met

  2. Power off the ZedBoard

  3. Connect EVAL-ADAQ7980-SDZ to the FMC LPC connector

  4. Connect USB cable from ZedBoard to host PC

  5. Connect signal source to EVAL-ADAQ7980-SDZ SMA input

  6. Set boot mode jumpers for JTAG boot:

    • MIO6: GND

    • MIO5: GND

    • MIO4: GND

    • MIO3: GND

    • MIO2: GND

  7. Power on the ZedBoard

Building the HDL Project

The HDL project must be built before the no-OS software.

Clone the HDL Repository

cd /path/to/workspace
git clone https://github.com/analogdevicesinc/hdl.git
cd hdl

Check the HDL Release Notes for the recommended Vivado version.

Configure Sampling Rate (Optional)

The default sampling rate is 1 MSPS. To change it, edit the configuration file:

nano projects/adaq7980_sdz/zed/system_bd.tcl

Modify the sampling rate parameter:

set adc_sampling_rate 1000000

Replace 1000000 with your desired sampling rate in Hz.

Build the Project

Build the HDL project for ZedBoard:

cd projects/adaq7980_sdz/zed
make

The build process will:

  1. Generate IP cores

  2. Create the block design

  3. Synthesize the design

  4. Implement the design

  5. Generate the bitstream

This may take 20-40 minutes depending on your system. The output bitstream will be located at:

projects/adaq7980_sdz/zed/adaq7980_sdz_zed.sdk/system_top.bit

Building the No-OS Software

After the HDL build completes, build the no-OS software.

Clone the No-OS Repository

cd /path/to/workspace
git clone https://github.com/analogdevicesinc/no-OS.git
cd no-OS

Set Up the Build Environment

Ensure the ARM cross-compiler is installed and in your PATH:

arm-none-eabi-gcc --version

If not installed, follow the No-OS Build Guide for setup instructions.

Configure the Project

The project uses the hardware description from the HDL build. Set the hardware platform path:

export HARDWARE=/path/to/hdl/projects/adaq7980_sdz/zed/adaq7980_sdz_zed.sdk

Or edit the Makefile to point to your HDL build location.

Build the Software

Build the no-OS project:

cd projects/adaq7980_sdz
make PLATFORM=xilinx ARCH=ARM

The build output will be:

projects/adaq7980_sdz/build/adaq7980_sdz.elf

Running the Project

Program the FPGA and Run No-OS

Use the Xilinx tools to program the FPGA and load the software:

Option 1: Using Vivado Hardware Manager

  1. Open Vivado

  2. Click “Open Hardware Manager”

  3. Click “Open Target” → “Auto Connect”

  4. Right-click on the FPGA device → “Program Device”

  5. Select the bitstream: system_top.bit

  6. Click “Program”

  7. After programming, right-click on the ARM core

  8. Select “Reset” → “Processor Reset”

  9. Click “Run” → “Add Configuration”

  10. Browse to adaq7980_sdz.elf

  11. Click “Run”

Option 2: Using Vitis (for newer Xilinx tools)

  1. Launch Vitis

  2. Create a new application project

  3. Select “Create a new platform from hardware (XSA)”

  4. Browse to the XSA file from the HDL build

  5. Import the no-OS source files

  6. Build and run

Option 3: Using XSCT Command Line

xsct
connect
targets -set -filter {name =~ "ARM*#0"}
rst -processor
fpga /path/to/system_top.bit
targets -set -filter {name =~ "ARM*#0"}
dow /path/to/adaq7980_sdz.elf
con

Verify Operation

Open a serial terminal (115200 baud, 8N1) to view the console output:

# Linux
screen /dev/ttyUSB0 115200

# Windows - use PuTTY or Tera Term
# COM port: varies by system
# Baud: 115200, Data: 8, Parity: None, Stop: 1

You should see initialization messages and ADC sample data.

Expected Output

The console should display:

ADAQ7980 Initialization...
ADAQ7980 successfully initialized
SPI Engine initialized
Starting continuous conversion...
Sample 0: 0x8000
Sample 1: 0x8001
Sample 2: 0x7FFF
...

The sample values will vary based on your input signal.

Testing and Validation

Basic Functionality Test

  1. Apply a DC voltage to the SMA input (within the input range)

  2. Verify the ADC readings correspond to the input voltage

  3. Apply a sine wave and verify the readings change accordingly

  4. Use the FFT analysis (if available) to verify signal quality

Configuration Options

The no-OS project supports various configuration options that can be modified in the source code:

  • Sampling rate (set in HDL)

  • Data output format

  • Averaging modes

  • Power-down modes

Refer to the no-OS project source and driver documentation for details.

Troubleshooting

FPGA Programming Issues

Problem: Cannot detect FPGA

Solution: - Check USB cable connection - Verify drivers are installed (Xilinx Cable Drivers) - Check Device Manager (Windows) or lsusb (Linux) - Try different USB port

Problem: Bitstream programming fails

Solution: - Verify bitstream file is not corrupted - Check Vivado version matches HDL requirements - Re-build the HDL project - Check power supply to ZedBoard

No Serial Output

Problem: No output on serial console

Solution: - Verify correct COM port is selected - Check baud rate is 115200 - Ensure USB cable is connected - Verify UART drivers are installed - Try resetting the processor

Problem: Garbled output

Solution: - Check baud rate (should be 115200) - Verify terminal settings (8N1) - Check for ground loop issues

Incorrect ADC Readings

Problem: Readings don’t match input voltage

Solution: - Verify signal source is connected properly - Check input voltage is within range - Verify reference voltage configuration - Check for ground connection issues - Review driver configuration

Problem: Noisy readings

Solution: - Check signal source impedance - Verify proper grounding - Check for ground loops - Ensure adequate power supply decoupling - Use shielded cables for analog input

Build Errors

Problem: HDL build fails

Solution: - Verify Vivado version matches requirements - Check IP core versions - Clean and rebuild: make clean && make - Review build log for specific errors

Problem: No-OS build fails

Solution: - Verify ARM toolchain is installed - Check HARDWARE path is correct - Ensure HDL build completed successfully - Review compiler errors and warnings

Next Steps

After successful setup:

  1. Explore the no-OS driver API

  2. Modify the application for your use case

  3. Experiment with different sampling rates

  4. Review the HDL documentation for customization options
