Beginner SDR Exercises

The following hands-on exercises demonstrate real-world software-defined radio applications using ADI’s SDR platforms. These exercises showcase the power of reconfigurable software running on specialized hardware, allowing you to implement complex signal processing algorithms without custom hardware design.

Hardware Platforms Used:

Throughout these exercises, you will work with three ADI SDR platforms, each offering different capabilities:

  • ADALM-PLUTO:

    Entry-level SDR ideal for learning and prototyping

    • 2x Tx and 2x Rx ports (50 Ohm matched)

    • LO Frequency Range: 70MHz → 6GHz

    • Bandwidth: 56MHz

    • Sample rate: 61.44MSPS; 14 bits

    • Interfaces: USB2, UART

  • Jupiter (AD-JUPITER-EBZ):

    Mid-range platform for advanced development

    • 2x Tx and 2x Rx ports (50 Ohm matched)

    • LO Frequency Range: 30MHz → 6GHz

    • Bandwidth: 40MHz

    • Sample rate: 61.44MSPS; 16 bits

    • Interfaces: USB3, 1Gb Ethernet, Display Port, UART

  • Talise SOM (ADRV9009-ZU11EG):

    High-performance platform for production-grade applications

    • 4x Tx and 4x Rx (expandable to 8 TRx)

    • LO Frequency Range: 75MHz → 6GHz

    • RX Bandwidth: 200MHz, TX Bandwidth: 450MHz

    • Interfaces: USB3, 1Gb Ethernet, Display Port, PCIe 3.0, SFP, QSFP, UART

All three platforms share the same software stack and HDL architecture, demonstrating ADI’s platform scalability—algorithms developed on the entry-level Pluto can be seamlessly transferred to production-grade hardware with minimal code changes.

Note

The following exercises should be run on a PC booted from the provided USB bootable image (available in the FTC2023 workshop Downloads section). The USB bootable image contains all necessary software pre-installed, including Python, GNU Radio, Thonny IDE, and all required libraries. Boot your PC from the USB drive with the mouse, keyboard, and monitor connected. Connect your SDR hardware (Pluto, Jupiter, or Talise) to the PC via USB before starting the exercises.

Transmit and receive a complex sinusoid (Sinewave Loopback)

A complex sinusoid is a fundamental building block in software-defined radio. Unlike a real sinusoid (which oscillates in a single dimension), a complex sinusoid consists of two components:

  • In-phase (I) component: Real part of the signal

  • Quadrature (Q) component: Imaginary part of the signal

Together, they form a single tone in the frequency domain. This exercise transmits a complex sinusoid and receives it back via a loopback connection, allowing you to observe both the transmitted signal and its frequency spectrum representation.

The following subsections demonstrate different implementations of this exercise using various tools and platforms.

Doppler Radar with GNU Radio (Pluto)

This exercise demonstrates the Doppler radar effect using GNU Radio Companion, a visual programming environment for building software-defined radio applications. You will create a continuous-wave (CW) radar system that can detect motion and measure relative velocity by observing frequency shifts in the received signal.

What is GNU Radio?

GNU Radio is a free, open-source software development toolkit that provides signal processing blocks to implement software-defined radios. It uses a visual flowgraph paradigm where signal processing blocks are connected together to create complete radio systems. GNU Radio Companion (GRC) provides a graphical interface for building these flowgraphs without requiring extensive programming knowledge.

Key features:

  • Visual flowgraph-based development environment

  • Extensive library of signal processing blocks

  • Hardware integration via gr-iio blocks for ADI devices

  • Real-time visualization and debugging tools

  • Python-based backend for custom block development

The Doppler Effect in Radar

The Doppler effect causes a frequency shift when there is relative motion between the radar and a target. When an object moves toward the radar, the reflected signal has a higher frequency (positive Doppler shift). When it moves away, the frequency decreases (negative Doppler shift).

Doppler Frequency Shift:

\[f_d = \frac{2 \cdot v \cdot f_c}{c}\]

where:

  • \(f_d\) = Doppler frequency shift

  • \(v\) = relative velocity of the target

  • \(f_c\) = carrier frequency

  • \(c\) = speed of light

How This Exercise Works:

  1. Transmit: The Pluto SDR transmits a continuous wave (CW) signal at 2.5 GHz

  2. Reflection: The signal reflects off your moving hand

  3. Receive: The Pluto receives the reflected signal with a Doppler-shifted frequency

  4. Processing: GNU Radio processes the signal to extract the Doppler shift

  5. Visualization: Real-time plots show the frequency shift, and audio output lets you hear the Doppler tone

What You Will Observe:

  • Moving your hand toward the antenna increases the frequency (higher pitch sound)

  • Moving away decreases the frequency (lower pitch sound)

  • The faster the motion, the larger the frequency shift

  • Real-time FFT plots visualize the Doppler shift

This is the same principle used in police speed radar guns, weather radar, and automotive collision avoidance systems.

  2. Open the terminal by simultaneously pressing Ctrl + Alt + T on your keyboard.

  3. Type the following command in the terminal to open GNU Radio Companion and press Enter:

Binary Phase Shift Keying (BPSK)

This exercise demonstrates Binary Phase Shift Keying (BPSK), a fundamental digital modulation technique used in modern communication systems. You will transmit digital data (binary bits) through SDR platforms, receive it back via a loopback connection, and observe how software-based signal processing recovers the original data despite real-world impairments.

What is BPSK?

Binary Phase Shift Keying is a type of phase modulation where digital information is encoded by shifting the phase of a carrier signal. In BPSK:

  • Bit 0 is represented by a phase of 180° (or -1 in the I/Q plane)

  • Bit 1 is represented by a phase of 0° (or +1 in the I/Q plane)

This creates two distinct constellation points on the real axis of the I/Q diagram, making BPSK one of the most robust digital modulation schemes against noise.

Where is BPSK Used?

BPSK and its variants are used in numerous communication systems:

  • GSM - Mobile phone networks

  • Satellite Television - DVB-S and other satellite broadcasting standards

  • Wi-Fi - Lower data rate modes (802.11b DSSS)

  • GPS and GNSS - Satellite navigation systems

  • RFID and NFC - Contactless communication

  • Deep space communications - Where robustness is critical

Challenges in Demodulating BPSK Signals

When receiving BPSK signals in real-world conditions, several impairments must be corrected:

  • Phase Offset: The local oscillator (LO) at the receiver may have a phase difference from the transmitter

  • Frequency Offset: Small differences in carrier frequency between TX and RX oscillators

  • Timing Offset: Clock differences require symbol timing recovery to sample at the right moment

  • Variation with time, distance, and temperature: These offsets change dynamically

As demonstrated in the previous Doppler radar exercise, frequency varies with distance and motion. These same effects impact digital communications.

The Power of Software-Defined Radio: All These Problems Can Be Solved by Software!

This exercise demonstrates how modern SDR systems correct all these impairments in software using sophisticated algorithms:

How the BPSK System Works:

Transmitter Chain:

  1. Configure SDR: Set center frequency, sample rate, and gain

  2. Create array of bits: Generate the digital data to transmit (0s and 1s)

  3. Interpolate and remap symbols: Repeat each bit 16 times for pulse shaping, map bit 0 → -1, bit 1 → +1

  4. Call tx() function: Transmit the modulated signal

Receiver Chain:

  1. Call rx() function: Receive the modulated signal

  2. Coarse frequency adjustment: Square the received signal to measure 2× frequency offset, then correct it

  3. Symbol timing recovery: Use Mueller and Muller clock recovery to select the correct sample from each 16-sample group

  4. Fine frequency and phase adjustment: Apply Costas loop to lock onto the correct phase and remove residual frequency offset

  5. Plot data: Visualize constellation diagrams and waveforms to observe signal quality

Key Signal Processing Techniques:

  • Frequency Offset Correction: The received signal is squared (raising phase errors to become visible as frequency offsets), an FFT identifies the peak at 2× the offset frequency, and samples are corrected by multiplying by \(e^{-j2\pi f_{offset} t}\)

  • Symbol Timing Recovery (Mueller and Muller): Since each bit is transmitted as 16 samples, the receiver must identify which of the 16 samples represents the optimal sampling point. The Mueller and Muller algorithm iteratively adjusts the timing offset to lock onto the correct samples.

  • Costas Loop (Fine Synchronization): Functions like a phase-locked loop (PLL) that multiplies I × Q samples. For ideal BPSK where Q = 0, this error function is minimized when there is no phase or frequency offset, allowing the loop to lock onto the correct carrier phase.

What You Will Observe:

Starting from a noisy, offset constellation with scattered points, you will see the signal progressively improve:

  1. Raw received signal: Shows significant phase and frequency offset

  2. After coarse frequency correction: Frequency offset mostly removed, but phase offset remains

  3. After symbol timing recovery: Correct samples selected, reducing mid-constellation points

  4. After Costas loop: Clean constellation with two tight clusters at -1 and +1

This exercise demonstrates the power of software-defined radio: complex signal processing algorithms running in software can correct real-world impairments that would be difficult or impossible to handle in analog hardware alone.

Quadrature Phase Shift Keying (QPSK)

This exercise demonstrates Quadrature Phase Shift Keying (QPSK), a digital modulation scheme that transmits 2 bits per symbol instead of BPSK’s 1 bit per symbol. QPSK uses four phase positions in the I/Q plane to encode data, effectively doubling the data rate in the same bandwidth.

What is QPSK?

QPSK encodes two bits into each symbol using four constellation points at 45°, 135°, 225°, and 315°:

  • 00, 01, 10, 11 → Four distinct phase positions

This creates twice the spectral efficiency of BPSK while maintaining good noise immunity.

Where is QPSK Used?

  • Satellite communications (DVB-S, GPS)

  • Wireless standards (CDMA, LTE, 5G)

  • Cable modems (DOCSIS)

  • Point-to-point microwave links

The following subsections demonstrate QPSK implementations ranging from basic reception to progressively more sophisticated receivers with full synchronization chains, showcasing how each processing block improves signal quality.

Simplified Signal Processing:

The transmitter configures the SDR, creates an array of bits, interpolates and remaps symbols, shifts the spectrum, and transmits the signal. Notably absent is the matched filter for pulse shaping.

The receiver receives the signal, shifts the spectrum back to baseband, and plots the data. It lacks the frequency correction, matched filtering, timing recovery, and phase correction that would normally be present in a proper QPSK receiver.

What This Demonstrates:

This exercise reveals several critical requirements for digital communication systems by intentionally omitting them. Without proper DSP blocks, you’ll observe:

  • Matched filters with pulse shaping for spectral efficiency

  • Differential encoding or preambles to resolve constellation ambiguity

Signal Processing with Matched Filters:

The transmitter configures the SDR, creates an array of bits, and uses a Constellation Modulator block that maps bytes into symbols and applies a Root-Raised Cosine filter for pulse shaping. The signal spectrum is then shifted and transmitted. The addition of matched filters at the transmitter optimizes bandwidth usage compared to the raw square wave spectrum observed in the previous exercise.

The receiver receives the signal, applies a matched Root-Raised Cosine filter to maximize signal-to-noise ratio, shifts the spectrum back to baseband, and plots the data. However, this implementation still lacks frequency offset correction, symbol timing recovery, and phase correction, demonstrating the progressive improvement in QPSK receivers.

Key Improvements Over Raw QPSK:

The most significant improvement is spectral efficiency. The Root-Raised Cosine matched filters at transmitter and receiver eliminate the high sidelobes seen in the raw QPSK implementation, concentrating signal energy within the necessary bandwidth. This reduces adjacent channel interference and allows more efficient spectrum usage.

The received signal quality improves due to the matched filter maximizing signal-to-noise ratio by correlating with the expected pulse shape. However, without symbol timing recovery, the receiver does not yet decimate by selecting the optimal samples from each symbol period.

What is Root-Raised Cosine (RRC) Filtering?

Root-Raised Cosine filtering is a pulse shaping technique that satisfies the Nyquist criterion for zero intersymbol interference. By using matched RRC filters at both transmitter and receiver, the combined response forms a complete raised cosine filter that eliminates interference between adjacent symbols while minimizing bandwidth.

The receiver now includes a critical addition to the signal processing chain. After receiving the signal through the matched RRC filter, a Frequency Locked Loop block dynamically adjusts the frequency of the received spectrum to compensate for oscillator differences between transmitter and receiver. The receiver then shifts the spectrum back to baseband and plots the data. However, fine frequency and phase adjustment using a Costas loop and symbol timing recovery for optimal sample selection are still missing from this implementation.

What is a Frequency Locked Loop?

A Frequency Locked Loop is a control system that automatically adjusts for coarse frequency offsets between transmitter and receiver local oscillators. Unlike a Phase Locked Loop which tracks both frequency and phase, an FLL focuses specifically on correcting frequency errors, making it useful as a first stage before fine synchronization with a PLL or Costas loop.

The FLL block continuously monitors the received spectrum and adjusts its center frequency to compensate for drift, ensuring the signal remains centered around DC (zero frequency) for subsequent processing stages.

Key Improvements Over Constellation Modulator:

The most significant improvement is automatic frequency offset correction. Without this correction, frequency differences between TX and RX oscillators would cause the constellation to rotate continuously, making reliable demodulation impossible. The FLL ensures the spectrum stays centered around DC, enabling subsequent processing stages.

However, the receiver still lacks fine frequency and phase correction that a Costas loop would provide, and symbol timing recovery to select the optimal samples from each symbol period. These will be added in more complete implementations.

The constellation shows improved stability compared to implementations without frequency correction, though fine frequency/phase correction (Costas loop) and symbol timing recovery would further improve quality.

Using GNU Radio with Pluto - Packet-Based File Storage

This exercise builds upon the previous QPSK demonstrations by implementing a complete packet-based communication system. Instead of displaying the received message on the console, this system receives QPSK-modulated data packets transmitted at regular intervals and extracts the message content to a file, demonstrating how real-world communication systems handle burst transmissions.

How Packet-Based Communication Works:

Unlike the continuous QPSK transmission you saw in previous exercises, this system transmits data in bursts (packets) at specific time intervals. Each packet contains:

  • Access Key (Preamble): A known synchronization pattern that helps the receiver detect the start of a packet

  • Payload Length: Information about how much data is in the packet

  • Message Data: The actual content being transmitted (e.g., “ANALOG DEVICES INC FTC2024”)

  • CRC32 Checksum: Error detection code to verify data integrity

Why Packet-Based Transmission?

Most real-world communication systems use packet-based transmission because:

  • Efficiency: Only transmit when you have data to send, saving power and bandwidth

  • Multiple Access: Allows multiple users to share the same frequency by taking turns

  • Error Detection: CRC32 checksum catches transmission errors

  • Synchronization: The receiver can lock onto each burst independently

GNU Radio Signal Flow:

Transmitter Chain:

  1. Message Source: Sends specific messages at regular time intervals

  2. Packet Formatter: Appends access key and payload length to each message

  3. CRC32: Appends checksum for error detection

  4. PDU to Tagged Stream: Converts packets (Protocol Data Units) to a continuous stream of data tagged with metadata

  5. QPSK Modulator: Modulates the data using Quadrature Phase Shift Keying

  6. SDR Sink: Transmits the modulated signal through the antenna

Receiver Chain:

  1. SDR Source: Receives the RF signal from the antenna

  2. QPSK Demodulator: Demodulates the signal to recover the symbol stream (similar to the previous exercise with frequency/phase/timing recovery)

  3. Packet Extractor: Extracts the message from the received stream of bits by detecting the access key, reading payload length, extracting data bytes, and verifying CRC32

  4. File Sink: Stores the decoded message to a file instead of displaying on console

Key Observation - Burst Mode Effects:

When you observe the constellation diagram for this exercise, you’ll notice that the symbols appear noisier compared to the continuous QPSK transmission. This is because:

  • Data is sent in bursts rather than continuously

  • The receiver must re-synchronize for each packet

  • Between bursts, there’s no signal, creating transitions in the constellation

  • The Costas loop and timing recovery have less time to stabilize for each burst

This is normal behavior for packet-based systems and demonstrates the trade-off between efficiency (burst mode) and signal quality (continuous mode).

Amplitude Shift Keying (ASK) in GNU Radio using Jupiter

This exercise demonstrates Amplitude Shift Keying (ASK), a digital modulation technique where information is encoded by varying the amplitude of a carrier signal. Unlike BPSK which varies the phase, or FSK which varies the frequency, ASK varies the signal strength to represent different symbols.

What is ASK?

Amplitude Shift Keying is a form of amplitude modulation where digital data is represented by changes in the amplitude of a carrier wave:

  • Different amplitude levels represent different symbols

  • In this exercise, we use 4-ASK (4 amplitude levels): 0, 1/3, 2/3, and 1

  • Each symbol represents 2 bits of data (00, 01, 10, 11)

  • The carrier phase remains constant while amplitude varies

Where is ASK Used?

ASK and its variants are used in various communication systems:

  • RFID tags - Simple, low-power devices

  • Optical fiber communications - On-Off Keying (OOK) is a binary form of ASK

  • Infrared remote controls - Simple amplitude modulation

  • Combined with PSK - Forms QAM (Quadrature Amplitude Modulation) used in Wi-Fi, LTE, 5G, cable modems

How This Exercise Works:

This GNU Radio flowgraph demonstrates 4-level ASK transmission and reception with the following signal processing:

Transmitter Chain:

  1. Vector Source: Generates four symbols repeatedly: [0, 1/3, 2/3, 1]

  2. Repeat Block: Repeats each symbol 100 times (100 samples per symbol) to create a visible waveform

  3. Multiply by Complex Sinusoid: Offsets the frequency of the transmitted signal relative to the TX LO frequency

  4. SDR Sink (Jupiter): Transmits the amplitude-modulated signal

Receiver Chain:

  1. SDR Source (Jupiter): Receives the RF signal from the loopback connection

  2. Complex to Magnitude: Extracts only the amplitude information from the received I/Q signal (discards phase)

  3. Keep 1 in N (Decimation): Keeps only 1 sample out of every 100 (recovers the original symbol rate)

  4. GUI Displays: Shows constellation, spectrum, time-domain waveforms, and decimated output

Key Observations:

  • Constellation Diagram: You will observe 4 distinct amplitude levels on the real axis (0, 1/3, 2/3, 1)

  • Magnitude RX Plot: Shows the varying amplitude levels in the time domain

  • Decimated RX: After decimation, you see the original symbol sequence recovered

  • Phase Information Discarded: Unlike BPSK/QPSK, only the magnitude matters in ASK, making it simpler but more susceptible to noise

From ASK to QAM - The Next Step:

When you combine Amplitude Shift Keying (ASK) with Phase Shift Keying (PSK), you get Quadrature Amplitude Modulation (QAM). QAM is one of the most widely used modulation schemes in modern communications because it achieves high data rates by encoding information in both amplitude and phase:

  • 16-QAM: 4 amplitude levels × 4 phase positions = 16 symbols (4 bits per symbol)

  • 32-QAM: 32 symbols (5 bits per symbol)

  • 64-QAM: 64 symbols (6 bits per symbol) - used in Wi-Fi 802.11n/ac

  • 256-QAM: 256 symbols (8 bits per symbol) - used in Wi-Fi 6, cable modems, LTE Advanced

Frequency Shift Keying (FSK) in GNU Radio using Jupiter

This exercise demonstrates Frequency Shift Keying (FSK), a digital modulation technique where information is encoded by varying the frequency of a carrier signal. Unlike BPSK which varies the phase, or ASK which varies the amplitude, FSK switches between different frequencies to represent different symbols.

What is FSK?

Frequency Shift Keying is a form of frequency modulation where digital data is represented by discrete frequency shifts of a carrier wave:

  • Different frequencies represent different symbols

  • In this exercise, we use 4-FSK (4 distinct frequencies)

  • Each symbol represents 2 bits of data (00, 01, 10, 11)

  • The amplitude and phase remain relatively constant while frequency changes

Where is FSK Used?

FSK and its variants are widely used in communication systems:

  • Caller ID - Phone systems use FSK to transmit caller information

  • Radio Telemetry - Weather stations, remote sensors

  • Pagers - POCSAG and FLEX paging protocols use FSK

  • Bluetooth - Uses Gaussian FSK (GFSK) for robust communication

  • LoRa - Long-range IoT uses Chirp Spread Spectrum (CSS), a continuous form of FSK

  • Older modems - Bell 103 and 202 modems used FSK at 300-1200 baud

  • APRS - Automatic Packet Reporting System for amateur radio

Advantages of FSK:

  • Noise resilience - Frequency changes are less affected by amplitude noise

  • Constant envelope - Power remains constant, efficient for power amplifiers

  • Simple demodulation - Can be demodulated with simple discriminators

  • Works well in fading channels - Better performance than ASK in multipath environments

How This Exercise Works:

This GNU Radio flowgraph demonstrates 4-level FSK transmission and reception with the following signal processing:

Transmitter Chain:

  1. Vector Source: Generates four symbols repeatedly representing different frequencies

  2. Repeat Block: Repeats each symbol to create longer frequency bursts

  4. Multiply Const: Adjusts the frequency deviation (how far apart the frequencies are)

  5. SDR Sink (Jupiter): Transmits the frequency-modulated signal

Receiver Chain:

  1. SDR Source (Jupiter): Receives the RF signal from the loopback connection

  2. Quadrature Demod: Converts frequency changes to amplitude changes (FM demodulation)

  3. Keep 1 in N (Decimation): Recovers the original symbol rate

  • Time Domain: The received signal shows frequency transitions between symbols

  • Waterfall Display: Shows the frequency hopping pattern over time

  • Demodulated Output: After quadrature demodulation, you see the original symbol sequence recovered

From FSK to More Advanced Modulations:

FSK represents one dimension of modulation (frequency). Modern communication systems combine multiple dimensions:

  • FSK + Time: Frequency-hopping spread spectrum (FHSS) used in Bluetooth

  • Continuous FSK: Gaussian FSK (GFSK) smooths transitions to reduce bandwidth

  • Chirp Spread Spectrum (CSS): Continuously sweeps frequency, used in LoRa for long-range IoT

  • OFDM: Uses many closely-spaced FSK-like subcarriers, foundation of Wi-Fi and LTE
