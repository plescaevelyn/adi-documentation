Beginner SDR Exercises

The following hands-on exercises demonstrate real-world software-defined radio applications using ADI’s SDR platforms. These exercises showcase the power of reconfigurable software running on specialized hardware, allowing you to implement complex signal processing algorithms without custom hardware design.

Note

The following exercises should be run on a Raspberry Pi 5 using the provided ADI Kuiper 2.0 Linux image. The Kuiper image contains all necessary software pre-installed, including Python, GNU Radio, Thonny IDE, and all required libraries. Connect your SDR hardware (Pluto, Jupiter, or Talise) to the Raspberry Pi 5 before starting the exercises.

Familiarize Yourself with the Tools

Before diving into the exercises, we recommend familiarizing yourself with the SDR hardware and tools you’ll be using. Understanding your hardware’s capabilities, features, and interfaces will help you get the most out of these exercises.

Explore the Hardware:

  • ADALM-Pluto (PlutoSDR) - Learn about the Pluto SDR platform, including hardware specifications, software drivers, and getting started guides.

These resources provide important information about:

  • Hardware specifications and capabilities

  • Supported frequency ranges and bandwidth

  • Software drivers and libraries (PyADI-IIO, GNU Radio, etc.)

  • Connectivity options and interfaces

  • Calibration and performance characteristics

Taking time to understand your hardware will make the exercises more meaningful and help you troubleshoot issues more effectively.

Transmit and receive a complex sinusoid (Sinewave Loopback)

A complex sinusoid is a fundamental building block in software-defined radio. Unlike a real sinusoid (which oscillates in a single dimension), a complex sinusoid consists of two components:

  • In-phase (I) component: Real part of the signal

  • Quadrature (Q) component: Imaginary part of the signal

Together, they form a single tone in the frequency domain. This exercise transmits a complex sinusoid and receives it back via a loopback connection, allowing you to observe both the transmitted signal and its frequency spectrum representation.

The following subsections demonstrate different implementations of this exercise using various tools and platforms.

Doppler Radar with GNU Radio (Pluto)

The Doppler effect causes a frequency shift when there is relative motion between radar and target. This exercise creates a continuous-wave radar using GNU Radio Companion.

The Pluto transmits a CW signal at 2.5 GHz. When the signal reflects off your moving hand, the received frequency shifts - higher pitch when moving toward, lower when moving away.

Follow these steps:

  1. Make the following setup using Pluto and connect it to your PC:

  2. Open the terminal by simultaneously pressing Ctrl + Alt + T on your keyboard.

  3. Type the following command in the terminal to open GNU Radio Companion and press Enter:

    analog@analog:~$

    gnuradio-companion

  4. In gnuradio-companion, click on the File->Open, and open from Desktop->workshop_exercises->gnuradio the doppler_radar_pluto.grc file.

  5. To run the flowgraph, press on the green arrow from the top function bar as indicated below.

#. Move your hand toward and away from the antennas and observe how the sound and the frequency of the received harmonic changes in real time plots.

The six plots show the received signal at different processing stages: top row displays time-domain waveforms, while the bottom row shows frequency-domain FFT plots where you can see the Doppler shift peak move left (hand moving away) or right (hand moving toward) from the center frequency.

Binary Phase Shift Keying (BPSK)

BPSK encodes digital data by shifting the carrier phase: bit 0 → -1, bit 1 → +1. This creates two constellation points on the real axis.

Real-world reception introduces frequency offset, phase offset, and timing errors. The receiver corrects these using:

  • Coarse frequency correction - squares the signal to find 2× offset, then compensates

  • Symbol timing recovery - Mueller and Muller algorithm selects optimal sample from each 16-sample symbol

  • Costas loop - fine frequency and phase correction

Quadrature Phase Shift Keying (QPSK)

QPSK encodes 2 bits per symbol using four constellation points at 45°, 135°, 225°, and 315°. This doubles spectral efficiency compared to BPSK.

Using GNU Radio with Jupiter - Without Additional DSP

QPSK without proper DSP - no matched filter, no frequency correction, no timing recovery. Shows why these blocks are essential: inefficient spectrum, constellation rotation, poor timing.

Follow these steps:

  1. Make the hardware setup using Jupiter SDR with RX1 and TX1 connected via SMA loopback cable and connect it to your PC.

  2. Open the terminal and navigate to the Desktop directory, then launch GNU Radio Companion with elevated privileges:

    analog@analog:~$

    cd /home/analog/Desktop

    analog@analog:~/Desktop$

    sudo ./start-grc.sh

  3. In GNU Radio Companion, open the following file from File → Open:

    analog@analog:~$

    /home/analog/Desktop/ftc_2024/10_qpsk_raw_loopback_gnuradio/QPSK_raw_loopback_jupiter.grc

  4. Observe the simplified flowgraph showing transmitter and receiver without DSP correction blocks.

  5. Click the Execute button (green arrow) to run the flowgraph.

  6. Observe the output displays. The transmitted spectrum shows wide square wave characteristics with high sidelobes. The received constellation displays scattered points with phase offset instead of clean clusters at the ideal symbol positions. The received spectrum demonstrates wide bandwidth due to the absence of pulse shaping.

  7. Click the Stop button (red square) when finished.

Using GNU Radio with Jupiter - Constellation Modulator

Adds Root-Raised Cosine (RRC) matched filters for pulse shaping. Improves spectral efficiency over raw QPSK, but still lacks timing recovery and phase correction.

Follow these steps:

  1. Make the hardware setup using Jupiter SDR with RX1 and TX1 connected via SMA loopback cable and connect it to your PC.

  2. Open the terminal and navigate to the Desktop directory, then launch GNU Radio Companion with elevated privileges:

    analog@analog:~$

    cd /home/analog/Desktop

    analog@analog:~/Desktop$

    sudo ./start-grc.sh

  3. In GNU Radio Companion, open the following file from File → Open:

    analog@analog:~$

    /home/analog/Desktop/ftc_2024/12_constellation_modulator_loopback_gnuradio/constellation_modulator_loopback_jupiter.grc

  4. Observe the flowgraph structure showing the Constellation Modulator block and matched RRC filters.

    The Constellation Modulator block in the transmitter maps bytes into symbols and applies the RRC filter in a single operation. The receiver includes a separate RRC filter block to complete the matched filter pair.

  5. Click the Execute button (green arrow) to run the flowgraph.

  6. Observe the output displays and compare to the previous raw QPSK exercise.

    The transmitted spectrum shows significantly improved bandwidth efficiency with minimal sidelobes due to RRC pulse shaping. Spectrum Paint using Pluto

This exercise demonstrates a creative application of SDR: “painting” images in the frequency spectrum. By transmitting signals at different frequencies over time, you can create visible patterns on a waterfall display, which shows frequency on the horizontal axis and time on the vertical axis.

How Spectrum Paint Works:

A waterfall display is a spectrogram that shows:

  • Frequency (horizontal axis): Different colors represent signal strength at each frequency

  • Time (vertical axis): Each new row represents a new time snapshot

  • Amplitude (color intensity): Brighter colors indicate stronger signals

By carefully controlling which frequencies are transmitted at each moment, you can “draw” images, text, or patterns that become visible as the waterfall scrolls down.

Applications:

While spectrum paint is primarily a demonstration, the underlying technique is used in:

  • Frequency-hopping spread spectrum (FHSS)

  • Chirp radar systems

  • Multi-carrier communications

  • Signal identification and watermarking

This exercise showcases the flexibility of software-defined radio: the same hardware can transmit any arbitrary frequency pattern by simply changing the software.
