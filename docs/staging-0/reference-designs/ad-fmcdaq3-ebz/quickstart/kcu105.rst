AD-FMCDAQ3-EBZ Kintex UltraScale KCU105 Quick Start Guide
=========================================================

This guide provides some quick instructions (still takes awhile to download, and
set things up) on how to setup the AD-FMCDAQ3-EBZ on :xilinx:`KCU105 <KCU105>`.

Prerequisites
-------------

Required Hardware
~~~~~~~~~~~~~~~~~

- :xilinx:`Xilinx KCU105 <KCU105>` Rev 1.0 or later board
-
  :adi:`AD-FMCDAQ3-EBZ <en/design-center/evaluation-hardware-and-software/evaluation-boards-kits/eval-fmcdaq3-ebz.html>`
  FMC board
- Ethernet cable
- 2 x Micro-USB cables

Required Software
~~~~~~~~~~~~~~~~~

- You need a Host PC (Windows or Linux)
- A UART terminal (Putty/Tera Term/Minicom, etc.), Baud rate 115200 (8N1)
- Xilinx Vivado 2019.1 or later - Xilinx Software Commandline Tool(XCST)
- Bitfile and Linux ELF image
- IIO Scope
  `Download <https://github.com/analogdevicesinc/iio-oscilloscope/releases>`__

.. tip::

   :dokuwiki:`Pre-built Images for KCU105. </resources/tools-software/linux-drivers/platforms/microblaze_loading>`

Setting up the hardware (KCU105)
--------------------------------

You will need to:

::

   -Get the [[xilinx>KCU105 | Xilinx KCU105]] board

   {{ resources:eval:user-guides:ad-fmcdaq3-ebz:quickstart:kcu105_daq3.jpg?500 }}

::

   -Connect the AD-FMCDAQ3-EBZ FMC board to the FPGA carrier **FMC1** HPC socket
   (**J22**)
   -Connect USB UART J4 (Micro USB) and USB JTAG J87 (Micro USB) to your host PC
   -Make sure you have pin 6 of SW15 on 0 and pin 5 on 0 to be able to use the
   board with JTAG (see
   [[xilinx>support/documents/boards_and_kits/kcu105/ug917-kcu105-eval-bd.pdf|UG917 - KCU105 Board User Guide]] > FPGA Configuration section)
   -Plug the Power Supply into 12V Power input connector (J15)
   -Turn it on

Programming the FPGA
--------------------

There are two ways of loading the design.

One is using the XCST command line, and the second method is to run the Tcl
script which takes care of loading the bit file and the Linux image.

If **no target** is shown, then check the following:

#. Check if your cable is connected to JTAG and is a data cable and not just a
   power-giving cable
#. Check if you have the cable drivers installed. To do so, run this script with
   *Administrator* rights C:\\Xilinx\\Vivado\\\
   *your_version*\\data\\xicom\\cable_drivers\\nt64\\\ ``install_drivers.cmd``
   and then check the log created in the same folder
   ``install_drivers_wrapper.log``. If everything is successful here (no
   errors), then you"re good to go. More details can be found
   `here in this thread <https://support.xilinx.com/s/question/0D52E00006hpN92SAE/trouble-with-cable-drivers-vivado-20201?language=en_US#:~:text=Right%20click%20on%20%22install_digilent.exe%22%20and%20%22Run%20as%20administrator%22.%20Make%20sure%20at%20this%20point%20that%20the%20cable%20is%20DISCONNECTED%20from%20the%20machine.%C2%A0>`__
   and
   `here in this Xilinx article <https://support.xilinx.com/s/article/1013230?language=en_US>`__.

2. Loading the design using tcl script
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Run Vivado TCL Shell from Windows start menu → Xilinx Design Suite → Vivado →
Vivado TCL Shell. (In Linux, source the settings.sh file first)

::

   ****** Vivado v2021.1 (64-bit)
     **** SW Build 3247384 on Thu Jun 10 19:36:33 MDT 2021
     **** IP Build 3246043 on Fri Jun 11 00:30:35 MDT 2021
       ** Copyright 1986-2021 Xilinx, Inc. All Rights Reserved.

   Vivado% cd /to/path/where/you/have/the/files
   Vivado% xsct run.tcl

   ****** Xilinx hw_server v2021.1
     **** Build date : Jun 10 2021 at 20:17:23
       ** Copyright 1986-2021 Xilinx, Inc. All Rights Reserved.
   INFO: hw_server application started
   INFO: To connect to this hw_server instance use url: TCP:127.0.0.1:3121
   Vivado%

.. tip::

   In case of using older releases of VIVADO **xsct run.tcl** command is
   replaced by **xmd -tcl run.tcl**
