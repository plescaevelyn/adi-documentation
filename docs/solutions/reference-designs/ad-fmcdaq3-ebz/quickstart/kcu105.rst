KCU105 Quickstart
=================

This guide provides some quick instructions on how to setup the
:adi:`AD-FMCDAQ3-EBZ` on:

- :xilinx:`KCU105` on FMC HPC (J22) port

.. image:: ./../images/kcu105.png
   :width: 900

.. esd-warning::

Using Linux as software
-----------------------

Necessary files
~~~~~~~~~~~~~~~

The following files are needed for the system to boot:

- HDL boot image: ``system_top.bit``
- Linux simple image: ``simpleImage.kcu105_fmcdaq3``
- Programming script: ``run.tcl``

- Instructions on how to choose the boot files from the SD card can be found in
  the **Platform-Specific Manual Steps** section from here:
  :external+kuiper:ref:`hardware-configuration`.
- Instructions on how to manually build the boot files from source can be found
  here:

   - :ref:`linux-kernel microblaze`
   - :external+hdl:ref:`daq3` build documentation. More HDL build details at
     :external+hdl:ref:`build_hdl`.

Required software
~~~~~~~~~~~~~~~~~

- AMD Xilinx Vivado and Vitis (downloading Vitis from `here <https://www.xilinx.com/support/download/index.html/content/xilinx/en/downloadNav/vitis.html>`_
  will include Vivado as well) - AMD Xilinx Software Command-Line Tool (XSCT)
- A UART terminal (Putty/Tera Term/Minicom, etc.) with baud rate 115200 (8N1)

Required hardware
~~~~~~~~~~~~~~~~~

- AMD Xilinx :xilinx:`KCU105` Rev 1.0 FPGA board or later and its power supply
- :adi:`AD-FMCDAQ3-EBZ` FMC evaluation board
- Micro-USB cable (UART)
- LAN cable (Ethernet)
- (Optional) 2x SMA cable for analog signal loopback

.. note::

   Pre-built images and loading instructions can be found at
    :dokuwiki:`Microblaze Linux boot <resources/tools-software/linux-drivers/platforms/microblaze_loading>`.

Testing
~~~~~~~

Creating the setup
^^^^^^^^^^^^^^^^^^

.. image:: ./../images/kcu105_daq3.jpg
   :width: 900

.. caution::

  This project was tested with **VADJ = 1.8 V**

In the following example, we will make a physical loopback between the ADC and
the DAC channels on the evaluation board, using SMA cables.

Follow the steps in this order, to avoid damaging the components:

#. Get the - :xilinx:`KCU105` board
#. Connect the :adi:`AD-FMCDAQ3-EBZ` FMC board to the FPGA carrier, on the
   **FMC1 HPC socket (J22)**
#. Connect USB UART J4 (Micro USB) to your host PC
#. Connect USB JTAG J87 (Micro USB) to your host PC
#. Configure the KCU105 for JTAG boot: set pin 6 of SW15 to **0** and pin 5 to
   **0** (refer to the :xilinx:`UG917 - KCU105 Board User Guide <support/documents/boards_and_kits/kcu105/ug917-kcu105-eval-bd.pdf>`,
   FPGA Configuration section)
#. Plug the Power Supply into the 12V Power input connector (J15)
#. Turn on the power switch on the FPGA board

.. esd-warning::

Programming the FPGA
^^^^^^^^^^^^^^^^^^^^

There are two ways of loading the design. One is using the XCST command line,
and the second method is to run the tcl script which takes care of loading the
bit file and the Linux image.

If **no target** is shown, then check the following:

- Check if your cable is connected to JTAG and is a data cable and not just a
  power-giving cable
- Cable drivers are installed. On Windows, run
  ``C:\Xilinx\Vivado\<version>\data\xicom\cable_drivers\nt64\install_drivers.cmd``
  as Administrator

1. Loading the design using XSCT command line
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

In Windows, run the XSCT terminal as **Administrator** from start menu → AMD Xilinx
Design Tools → AMD Xilinx Software Command Line Tool → Run as Administrator.
