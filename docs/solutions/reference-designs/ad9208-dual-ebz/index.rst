AD9208-DUAL-EBZ User Guide
===========================

Introduction
------------

The AD9208-DUAL-EBZ reference design is a processor-based (Microblaze)
embedded system featuring two :adi:`AD9208` dual-channel ADCs. The
:adi:`AD9208` is a dual, 14-bit, 3 GSPS analog-to-digital converter (ADC)
designed for wide bandwidth applications. The device interfaces to the FPGA
transceivers followed by the individual JESD204B and ADC cores. The cores
are programmable through an AXI-Lite interface. The samples are passed to
the system memory (DDR).

The ADCs are set to run at full bandwidth mode at 3 GSPS, which translates to
a lane rate of 15 Gbps. Each converter has its own SYSREF driven from a common
HMC7044 clock chip. Sampling clocks are generated by the same clock chip.

-----------------

- :adi:`AD9208`

Supported Carriers
------------------

- :xilinx:`VCU118 <products/boards-and-kits/vcu118.html>` FMC+ Slot

HDL Reference Design
--------------------

The design has two JESD receive chains each having 8 lanes at a rate of
15 Gbps. The JESD receive chain consists of a physical layer represented by
an XCVR module, a link layer represented by an RX JESD LINK module, and a
transport layer represented by a RX JESD TPL module.

The links operate in Subclass 1 using the SYSREF signal to edge-align the
internal local multiframe clock and to release the received data from all
lanes simultaneously. This ensures that data from all channels is
synchronized at the application layer.

Both transport layer components present 256 bits on their output on every
clock cycle, representing 8 samples per converter. The two receive chains
are merged together and transferred to the DDR with a single DMA. An ADC
buffer stores 65k samples per converter in the fabric before transferring
with the DMA.

JESD204B Configuration
~~~~~~~~~~~~~~~~~~~~~~

Both links are set for full bandwidth mode and operate with the following
parameters:

.. list-table::
   :header-rows: 1

   * - Parameter
     - Value
   * - Deframer (L, M, F, S, N')
     - L=8, M=2, F=1, S=2, N'=16
   * - GLBLCLK
     - 375 MHz (Lane Rate / 40)
   * - REFCLK
     - 750 MHz (Lane Rate / 20)
   * - SYSREF
     - 1.46 MHz (DEVCLK / 2048)
   * - DEVCLK
     - 3000 MHz
   * - JESD204B Lane Rate
     - 15 Gbps

Quick Start Guide
-----------------

.. figure:: hardware_setup.png
   :align: center

   AD9208-DUAL-EBZ hardware setup on VCU118

Hardware Requirements
~~~~~~~~~~~~~~~~~~~~~

- :xilinx:`VCU118 <products/boards-and-kits/vcu118.html>` Rev 1.0 or later board
- AD9208-DUAL-EBZ FMC board
- 2 x Micro-USB cable
- Ethernet cable
- Signal generator

Board Power
~~~~~~~~~~~

There are two options for powering the AD9208-DUAL-EBZ board:

- Populate fuse F1 with a 2 A fuse to supply the board from the FMC connector
  (requires the FMC carrier to drive 2 A on its 12 V power rail).
- Use an external 12 V power supply connected to TP1 (PWR_IN) and TP2 (AGND).

Software Requirements
~~~~~~~~~~~~~~~~~~~~~

- A Linux host PC
- Xilinx Vivado 2019.1 or later
- UART terminal (e.g. TeraTerm)
- :doc:`IIO Oscilloscope </software/iio-oscilloscope/index>`

Building the HDL Project
~~~~~~~~~~~~~~~~~~~~~~~~

.. code-block:: bash

   cd hdl/projects/ad9208_dual_ebz/vcu118
   make

For general build prerequisites and instructions, refer to the
`Building HDL projects <https://analogdevicesinc.github.io/hdl/user_guide/build_hdl.html>`__
guide.

Building the Linux Kernel
~~~~~~~~~~~~~~~~~~~~~~~~~

The design runs MicroBlaze Linux. Build the kernel and devicetree with:

.. code-block:: bash

   make simpleImage.vcu118_dual_ad9208

For instructions on building MicroBlaze Linux from source, see
:doc:`/linux/kuiper/index`.

Programming the Board
~~~~~~~~~~~~~~~~~~~~~

Gather the ``system_top.bit`` from the HDL build and the
``simpleImage.vcu118_dual_ad9208`` from the Linux build into a working
directory. Program the board using XSCT or a similar tool:

.. code-block:: bash

   xsct% connect
   xsct% fpga -f system_top.bit
   xsct% after 1000
   xsct% target 3
   xsct% dow simpleImage.vcu118_dual_ad9208.strip
   xsct% after 1000
   xsct% con
   xsct% disconnect

After the system boots, the following IIO devices should be present:

.. code-block:: bash

   # iio_info | grep iio:device
           iio:device0: hmc7044
           iio:device1: axi-ad9208-0-hpc (buffer capable)
           iio:device2: axi-ad9208-1-hpc

Capturing Data with IIO Oscilloscope
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

From the UART console, determine the IP address assigned by the DHCP
server (or manually assign one). Launch
:doc:`IIO Oscilloscope </software/iio-oscilloscope/index>` on the host
PC and connect to the board's IP address.

When the device is configured into **JESD Subclass 1** (default), the
two JESD links are synchronized and there is no phase difference between
the input channels.

When the device is configured into **JESD Subclass 0**, the two links
are not synchronized and phase differences between channels may be
observed.

Evaluating the AD9208 with ACE
-------------------------------

The :adi:`AD9208` can also be evaluated as a standalone device using the
AD9208-3000EBZ evaluation board paired with an FPGA-based data capture board
(ADS7-V2 or ADS8-V1) and the
`ACE <https://www.analog.com/en/resources/evaluation-hardware-and-software/evaluation-development-platforms/ace-software.html>`__
(Analysis, Control, Evaluation) software. This section covers the single-board
evaluation workflow.

Equipment Needed
~~~~~~~~~~~~~~~~

- PC running Windows with a USB 2.0 port
- AD9208-3000EBZ evaluation board
- ADS7-V2EBZ or ADS8-V1 FPGA-based data capture board
- 12 V, 6.5 A switching power supply (such as the SL POWER CENB1080A1251F01
  supplied with the ADS7-V2EBZ)
- USB 2.0 High-Speed A-to-B cable
- Low phase-noise analog input source with antialiasing filter
- Low phase-noise sample clock source
- Reference clock source

.. note::

   The selection of the FPGA data capture board depends on the required
   JESD204B lane rate. If the lane rate is 12.5 Gbps/lane or below, the
   ADS7-V2 can be used. For lane rates above 12.5 Gbps up to 16 Gbps/lane,
   the ADS8-V1 is required.

Connector and Jumper Configuration
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. figure:: board_connector_layout.png
   :align: center
   :width: 600

   AD9208-3000EBZ connector layout

.. figure:: board_jumpers.png
   :align: center
   :width: 600

   Jumper connections on the AD9208-3000EBZ

Before using the software for testing, configure the evaluation board as
follows:

1. Before connecting the AD9208 evaluation board to the ADS7-V2, install
   jumpers on **P304, P305, P307, P308, P309, P311**, and **P312**. Do not
   install jumpers on **P7** (Temperature Sensor Enable) or **P100**
   (Power Down/Standby). Install **P401** towards the inside of the board to
   power the board via FMC.
2. Ensure the data capture board power switch (**S1**) is set to OFF. Connect
   the evaluation board to the data capture board via the FMC connector on the
   underside of the board. Connect the power supply and USB cable to the
   data capture board.
3. Turn on the ADS7-V2EBZ. It should appear in the Windows Device Manager.
4. On the AD9208 evaluation board, provide a clean, low-jitter clock source
   to connector **J201** at 10 dBm. Set the sample clock frequency to
   3 GHz for the AD9208.
5. On the ADS7-V2, provide a clean, low-jitter reference clock to connector
   **J3** at 10 dBm. The REFCLK frequency is calculated as:

   ``REFCLK = Lane Rate / 20``

   where ``Lane Rate = (M * N' * (10/8) * f_out / L)``, and
   ``f_out = f_ADC_SAMPLE_CLOCK / Decimation Ratio``.

6. Connect a clean signal generator with low phase noise to **J101** or
   **J104** (Channel A or B respectively). Use a narrow-band bandpass filter
   with 50-ohm terminations.

ACE Software Setup
~~~~~~~~~~~~~~~~~~

1. Download and install
   `ACE <https://www.analog.com/en/resources/evaluation-hardware-and-software/evaluation-development-platforms/ace-software.html>`__
   if not already installed.
2. Download the AD9208 ACE plugin from the
   :adi:`AD9208 evaluation board software section <en/design-center/evaluation-hardware-and-software/evaluation-boards-kits/EVAL-AD9208.html#eb-relatedsoftware>`
   or install it via ACE's Plugin Manager (Tools > Manage Plug-Ins).
3. Launch ACE. The AD9208 plugin should appear on the start page.

.. figure:: ace_1.png
   :align: center
   :width: 200

   ACE AD9208 plugin icon

4. Click the plugin to open the Board View. Click **Program FPGA Image** to
   load the FPGA bitstream for communicating with the AD9208-3000EBZ.

.. figure:: ace_a1.png
   :align: center
   :width: 350

   ACE AD9208 Board View

.. warning::

   Programming the FPGA powers the AD9208 evaluation board via the FMC
   connector. Removing any of the board's power jumpers while the board is
   on or in operation may cause damage to the board, components, and/or
   the chip.

5. Double-click the blue AD9208 chip block in the Board View to open the
   Chip View.

Obtaining a Full Bandwidth Capture
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

1. Under Initial Configuration, change the number of Virtual Converters to 1
   and click **Apply** to apply the chip settings. Set the reference clock
   to the appropriate frequency as recommended by the ACE plugin.

.. figure:: ace_6.png
   :align: center
   :width: 250

   Data path reset checkbox

2. Issue a data path reset by clicking its checkbox and clicking
   **Apply Changes**. The data path reset bit automatically self-clears.
3. If the PLL Lock Lost indicator lights up, reset it by powering down the
   JESD link using the Link Control dropdown box, then re-enable the link
   and click **Apply Changes**.
4. Click **Proceed to Analysis**. This opens ACE's Analysis tool for ADC
   data, displaying both sample plots and FFTs. Click **FFT** and run one
   capture.

.. figure:: ace_15.png
   :align: center
   :width: 500

   AD9208-3000EBZ FFT at 255 MHz analog input to Channel A

5. To capture from Channel B, use the Register Debugger within ACE and write
   ``0x01`` to register ``0x0564``, then click **Read All** and repeat the
   capture.

Obtaining a DDC Capture
~~~~~~~~~~~~~~~~~~~~~~~~

The AD9208 supports Digital Down Converter (DDC) modes for decimation and
frequency translation:

1. Under Initial Configuration, set the Chip Operating Mode for two DDCs.
2. Select "HB1_HB2_HB3 Complex" for decimate-by-8 mode with three half-band
   filters.
3. Set the number of lanes to 4, converters to 4, and octets per frame to 2.
   Click **Apply**.
4. Click the NCO block to set the Numerically Controlled Oscillator frequency
   (e.g. 1300 MHz). Enable the 6 dB DDC gain from the dropdown menu. Click
   **Apply Changes**.
5. Navigate to the second DDC (DDC1) and make the same changes.
6. In the Analysis view, run a capture. DDC0 can be selected from Channel A
   and DDC1 from Channel B.

Troubleshooting
---------------

Board Not Functioning Properly
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

- Verify the evaluation board is properly seated in the FMC connector.
- Check the supply domain voltages while the board is powered:

.. list-table::
   :header-rows: 1

   * - Domain
     - Jumper
     - Test Point
     - Approx. Voltage
   * - AVDD_1
     - P307
     - TP303
     - 0.975 V
   * - AVDD_2
     - P308
     - TP304
     - 1.90 V
   * - AVDD_3
     - P309
     - TP305
     - 2.50 V
   * - DRVDD_1
     - P304
     - TP301
     - 0.975 V
   * - DRVDD_2
     - P311
     - TP306
     - 1.90 V
   * - DVDD
     - P305
     - TP302
     - 0.975 V
   * - SPI_VDD
     - P312
     - TP307
     - 1.90 V

- If a short is detected between any supply domain and ground, or an open is
  detected across fuse chip F401 (next to P401), a component may have been
  damaged from jumper or board removal while actively powered.

No SPI Communication
~~~~~~~~~~~~~~~~~~~~~~

- Verify the FPGA on the ADS7-V2 has been programmed (LED DS15 / FPGA_DONE
  should be lit and the fan should be running).
- Check the common mode voltage on the JESD204B traces. On the evaluation
  board, it should be approximately two-thirds of DRVDD_1. On the ADS7-V2,
  it should be around 1.2 V.
- Check Test Point 307 (SPI_VDD supply domain, jumper P312) and ensure it
  reads around 1.9 V.
- To test SPI operation, use ACE's Register Debugger to read and write
  register ``0x000A``. This is an open register available for testing memory
  reads and writes. If it reads back the same value written, SPI is
  operational.
- All registers reading back as ``0xFF`` or ``0x00`` may indicate no SPI
  communication.

Board Fails to Capture Data
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

- Ensure the board is functioning properly and SPI communication is
  successful (see above).
- Check the Clock Detect register ``0x011B``. A value of ``0x01`` indicates
  the sample clock is detected; ``0x00`` indicates no clock detected. Verify
  the signal generator input on connector J201.
- Check the PLL Locked indicator in the plugin chip view or register
  ``0x056F``. If the register reads ``0x80``, the PLL is locked. If not:

  - Verify the clock input to J201 (3 GHz for AD9208).
  - Verify the JESD204B settings under Initial Configuration match the
    :adi:`AD9208` datasheet specifications.
  - Verify the reference clock matches the JESD settings.
  - Ensure jumper P100 (Power Down/Standby) is not installed.
