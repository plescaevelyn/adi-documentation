Software
========

This page describes the demo software for the :ref:`eval-cn0394-ardz`
multi-channel thermocouple temperature measurement system running on the
EVAL-ADICUP360 platform.

General Description
-------------------

The **ADuCM360_demo_cn0394** is a temperature measurement demo project which
uses thermocouple measurement with the EVAL-ADICUP360 base board and the
EVAL-CN0394-ARDZ shield, created using the GNU ARM Eclipse Plug-ins in Eclipse
environment.

The EVAL-CN0394-ARDZ shield uses the internal ADC from the ADuCM360
microcontroller which has 2 channels available, 24-bit resolution with PGA and
reference. The first channel is used for RTD measurements, while the second
channel measures thermocouple changes.

.. figure:: cn0394_demo_1.png
   :width: 650px
   :align: center

   EVAL-CN0394-ARDZ connected to EVAL-ADICUP360.

The EVAL-CN0394-ARDZ board has 4 thermocouple ports (**P1** to **P4**) and 4
Pt1000 RTDs (**R1** to **R4**) close to each thermocouple socket for cold
junction compensation. It supports all 8 types of thermocouple: T, J, K, E, S,
R, N, B with full range temperature measurement. The RTD operates in 2-wire
mode and the excitation current is generated by the ADuCM360.

The application reads the 4 RTD channels and the 4 thermocouple channels,
processes them, and makes all necessary calculations to output a linearized
temperature for each available port. The UART interface (19200 baud rate, 1
start bit, 8-bit data length, no parity bits, and 2 stop bits) is used to send
data to a terminal window.

The user can enable/disable calibration on each channel separately. If
calibration is enabled, the user is informed when calibration is completed;
otherwise, the user is informed that calibration is disabled (see
``USE_RTD_CALIBRATION`` and ``USE_TH_CALIBRATION`` parameters).

Before starting the measurement, it is required to set up the thermocouple
types for each port (check ``P1_TYPE`` to ``P4_TYPE``). Based on those settings
the output data is calculated and displayed in the terminal window continuously,
considering the data refresh parameter (see ``DISPLAY_REFRESH``). Warning
messages are displayed if the final linearized temperature for the selected
thermocouple is out of the boundaries.

.. figure:: cn0391_demo_6.png
   :width: 750px
   :align: center

   Serial terminal output showing thermocouple temperature measurements.

Temperature Measurement and Linearization Algorithm
----------------------------------------------------

This project uses a *voltage to temperature* solution based on cold junction
temperature compensation and thermocouple measurements to provide a final
linearized temperature. The linearization is dictated by thermocouple type and
uses standard formulas generated by the
`National Institute of Standards and Technology (NIST)
<http://srdata.nist.gov/its90/download/download.html>`__.

When implementing thermocouple measurements, the thermocouple measures the
temperature relative to the cold junction. The cold junction temperature must
be added to give the overall temperature. Converting the measured voltage to a
temperature requires linearization using the cold junction temperature together
with the voltage generated by the thermocouple.

Cold Junction Measurement
~~~~~~~~~~~~~~~~~~~~~~~~~~

For cold junction temperature measurement, the **PT1000 RTD method** is used.
Cold junction reading from each RTD channel is used to calculate RTD
resistance:

.. figure:: cn0391_demo_5.png
   :width: 550px
   :align: center

   RTD resistance calculation formula.

The PT1000 RTD is ideally 1000 Ohm at 0 degrees C. There are two separate
equations used for linearization depending on the calculated resistance:

**1. RTD resistance > 1000 Ohm:**

.. figure:: cn0391_demo_3.png
   :width: 800px
   :align: center

   Callendar-Van Dusen equation for positive temperatures.

**2. RTD resistance <= 1000 Ohm:**

.. figure:: cn0391_demo_4.png
   :width: 800px
   :align: center

   Extended Callendar-Van Dusen equation for negative temperatures.

Thermocouple Measurement
~~~~~~~~~~~~~~~~~~~~~~~~~~

For thermocouple temperature linearization, standard
`NIST equations <http://srdata.nist.gov/its90/download/download.html>`__
are used. The thermocouple type used dictates the formulas needed for
linearization.

First, the cold junction temperature is converted to a cold junction voltage.
This conversion uses a standard polynomial formula. The formula is the same for
all thermocouple types; however, the coefficient values depend on the
thermocouple type.

.. figure:: cn0391_demo_7.png
   :width: 800px
   :align: center

   NIST polynomial formula for cold junction voltage conversion.

The final thermocouple voltage is calculated using the thermocouple voltage
read on the thermocouple channel and adding the converted cold junction
voltage:

.. code-block:: none

   Vtc = Vtc_read + Vcj

   Where:
     Vtc      = final voltage value for thermocouple channel
     Vtc_read = voltage value measured on thermocouple channel
     Vcj      = cold junction voltage calculated using NIST formula

For the final linearized temperature value, the second polynomial formula
provided by the NIST standard is used:

.. figure:: cn0391_demo_8.png
   :width: 800px
   :align: center

   NIST polynomial formula for final linearized temperature.

Demo Requirements
-----------------

Hardware
~~~~~~~~

- EVAL-ADICUP360
- EVAL-CN0394-ARDZ
- Up to 4 thermocouple sensors (Type T, J, K, E, S, R, N, B)
- Micro USB to USB cable
- PC or laptop with a USB port

Software
~~~~~~~~

- ADuCM360_demo_cn0394 software
- CrossCore Embedded Studio (2.7.0 or higher)
- ADuCM36x DFP (1.0.2 or higher)
- CMSIS ARM Pack (4.3.0 or higher)
- Serial terminal program (such as PuTTY or Tera Term)

Obtaining the Source Code
-------------------------

There are two ways to program the ADICUP360 with the CN0394 software:

1. **Drag and Drop** -- Drag and drop the .bin file to the MBED drive. This is
   the easiest way to get started.
2. **Build with CCES** -- Import the project into CrossCore Embedded Studio to
   build, compile, and debug. This allows parameter customization.

.. admonition:: Download

   Prebuilt CN0394 Bin File:

   - `ADuCM360_demo_cn0394.bin
     <https://github.com/analogdevicesinc/EVAL-ADICUP360/releases/download/Release-1.0/ADuCM360_demo_cn0394.bin>`__

   Complete CN0394 Source Files:

   - `ADuCM360_demo_cn0394 Source Code
     <https://github.com/analogdevicesinc/EVAL-ADICUP360/tree/master/projects/ADuCM360_demo_cn0394>`__

Configuring the Software Parameters
------------------------------------

The following parameters can be configured in the source code:

- **RTD channel calibration** -- ``USE_RTD_CALIBRATION`` parameter in
  ``CN0394.h``. Set to ``YES`` to enable calibration, ``NO`` to disable.
- **Thermocouple channel calibration** -- ``USE_TH_CALIBRATION`` parameter in
  ``CN0394.h``. Set to ``YES`` to enable calibration, ``NO`` to disable.
- **Terminal refresh** -- ``DISPLAY_REFRESH`` parameter in ``CN0394.h``. Sets
  how often to refresh the output data, in milliseconds.
- **Thermocouple types** -- ``P1_TYPE`` to ``P4_TYPE`` parameters in
  ``Thermocouple.h``. Select from ``TYPE_T`` through ``TYPE_B`` for each port.

Serial Terminal Output
----------------------

1. Flash the program to the EVAL-ADICUP360.
2. Switch the USB cable from the DEBUG USB (P14) to the USER USB (P13).
3. Configure the serial terminal with the following UART settings:

.. code-block:: none

   Select COM Port
   Baud rate: 19200
   Data: 8 bit
   Parity: none
   Start: 1 bit
   Stop: 2 bit
   Flow Control: none

Project Structure
-----------------

The **ADuCM360_demo_cn0394** is a C project that uses the ADuCM36x C/C++
Project structure.

.. figure:: cn0394_demo_3.png
   :width: 300px
   :align: center

   CN0394 demo project structure.

The project contains:

- System initialization -- disabling watchdog, setting system clock, enabling
  clock for peripherals
- ADC setup
- UART read/write functions
- Calculation algorithm for cold junction and linearized thermocouple
  temperature

In the **src** and **include** folders:

- ``Communication.c/h`` -- UART specific data
- ``CN0394.c/h`` -- Calculation part
- ``ADC.c/h`` -- ADC channels handling
- ``RTD.h`` -- RTD channel specific data for temperature calculation
- ``Thermocouple.h`` -- Thermocouple channel specific data for temperature
  calculation

The **RTE** folder contains device and system related files:

- **Device Folder** -- Low-level drivers for ADuCM360 microcontroller
- **system.rteconfig** -- Peripheral component selection, startup, and ARM
  CMSIS files
