ZCU102 Quick start
===============================================================================

This guide provides quick instructions on how to setup the
:adi:`X-Band Phased Array Platform <resources/evaluation-hardware-and-software/evaluation-boards-kits/x-band-development-platform>` on:

- :xilinx:`ZCU102`

.. image:: ../../images/zcu102.jpg
   :width: 900

.. esd-warning::

Using Linux as software
-------------------------------------------------------------------------------

Necessary files
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. note::

  The SD card includes several folders in the root directory of the BOOT
  partition. In order to configure the SD card to work with a specific FPGA
  board and ADI hardware, several files must be copied onto the root directory.
  Using the host PC, drag and drop the required files onto the BOOT partition,
  and use the EJECT function when removing the SD card from the reader.

The following files are needed for the system to boot:

- HDL boot image: ``BOOT.BIN``
- Linux Kernel image: ``Image``
- Linux device tree: ``system.dtb``

They can either be taken from the SD card -- already generated by us, or you
can build them manually:

- Instructions on how to choose the boot files from the SD card can be found in
  the **Platform-Specific Manual Steps** section from here:
  :external+kuiper:ref:`hardware-configuration`.

- Instructions on how to manually build the boot files from source can be found
  here:

   - :ref:`linux-kernel zynqmp`
   - :external+hdl:ref:`ad9081_fmca_ebz_x_band` build documentation. More HDL build details
     at :external+hdl:ref:`build_hdl`.

.. important::

  Some projects provide multiple devicetree files in the SD card's boot
  folders. Make sure you select the devicetree that matches your specific use
  case.

Required Software
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

- SD Card 16GB imaged with :external+kuiper:doc:`Kuiper <index>`
  (check out that guide on how to do it, then come back to this section)
- A UART terminal (Putty/Tera Term/Minicom, etc.) with baud rate 115200 (8N1)

Additional Software
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
- MATLAB

Creating the setup
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

.. image:: ../images/xbdp-system-block-diagram.png
   :width: 800

Follow the steps in this order, to avoid damaging the components:

#. Connect the :adi:`EVAL-AD9081` FMC board to the
   :xilinx:`ZCU102` HPC0 FMC socket as shown below:

   .. note::
      Don't forget to use the FMC Extender Board to raise the :adi:`EVAL-AD9081` up!

   .. image:: ../images/mxfe_connection.jpg
      :width: 400

#. Connect the :ref:`XUD1A<xud1a>` evaluation board to the :xilinx:`ZCU102`'s
   HPC1 port, using the FMC ribbon cable, interposer board, and 14 pin PMOD
   cable. The FMC ribbon cable is used both to move the interposer board and
   :ref:`XUD1A<xud1a>` away from the
   :adi:`MxFE <resources/evaluation-hardware-and-software/evaluation-boards-kits/eval-ad9081>`
   evaluation board as well as to allow access to the HPC1 connector which is
   otherwise blocked by the
   :adi:`MxFE <resources/evaluation-hardware-and-software/evaluation-boards-kits/eval-ad9081>`
   evaluation board

   .. note::
      The Interposer board PMOD is pin-compatible with the :ref:`XUD1A<xud1a>` PMOD enabling a direct connect.
      The 14-Pin PMOD 6 Inch Cable is optional, but recommended as it allows :ref:`XUD1A<xud1a>` to move away
      from the interposer board for easier RF cabling connections

   The digital connections between the :xilinx:`ZCU102` and the
   :ref:`XUD1A<xud1a>` are shown in the image below:

   .. image:: ../images/xud1a_fmc_v2.png
      :width: 800

#. Connect the :ref:`XUD1A<xud1a>` board to the :adi:`EVAL-AD9081`, using 8 SMPM
   to SMA cables and, optionally, RF Low Pass Filters, as indicated in the table
   below:

   .. note::
      The :ref:`XUD1A<xud1a>` contains IF bandpass filters per channel (FL3, FL6, FL7, FL8). The rejection for
      these filters is not sufficient for optimized system performance and additional rejection is
      required to suppress mixing products, LO feedthrough, DAC image, etc.

      - - Connection #
        - MxFE Connector (SMA)
        - (Optional) RF Filter

        - `VLF-8400+ <https://www.minicircuits.com/WebStore/dashboard.html?model=VLF-8400%2B>`_
        - J10
      - - 2
        - ADC1
        - `VLF-8400+ <https://www.minicircuits.com/WebStore/dashboard.html?model=VLF-8400%2B>`_

      - - 3

        - `VLF-5500+ <https://www.minicircuits.com/WebStore/dashboard.html?model=VLF-5500%2B>`_
        - J9
      - - 4

        - `VLF-5500+ <https://www.minicircuits.com/WebStore/dashboard.html?model=VLF-5500%2B>`_
        - J7
      - - 5

        - `VLF-5500+ <https://www.minicircuits.com/WebStore/dashboard.html?model=VLF-5500%2B>`_

   The connections between the :ref:`XUD1A<xud1a>`, :adi:`EVAL-AD9081`, and
   :xilinx:`ZCU102` boards should look similar to the image below:

   .. image:: ../images/ad9081-xud1a-connect.png
      :width: 800

#. Connect the :ref:`Stingray<stingray>` board to the PMOD connectors on the
   ZCU102, using the 12 pin PMOD cables and any required adapters, as described
   below:

   - J55 from the ZCU102 should connect to P3 on the :ref:`Stingray<stingray>`
     board
   - J87 from the ZCU102 should connect to P4 on the :ref:`Stingray<stingray>`
     board

   .. note::
      Pin 1 should connect to pin 1. This will require two PMOD cables to vertically flip the pinout from
      the ZCU102 to match that of the :ref:`Stingray<stingray>` board. Length depends on desired setup, but the total length
      of each combined PMOD cable will need to be at least 20“ (50cm). You can use any combination of 12-pin
      cable compatible with PMOD headers and 12 Pin 0.100” Male-Male Headers

   The digital connections between the :xilinx:`ZCU102` and the
   :ref:`Stingray<stingray>` are shown in the image below:

   .. image:: ../images/stingray_zcu102_pmod.png
      :width: 800

#. Connect the :ref:`Stingray<stingray>` to the :ref:`XUD1A<xud1a>`, using 8
   SMPM to SMA cables and 4 SMA to SMA 50 Ohm adapters or cables. The
   :ref:`XUD1A<xud1a>` has 4 RFIO ports whereas the :ref:`Stingray<stingray>`
   board has 8. For this system, the :ref:`Stingray's<stingray>` 8 channels will
   be paired using external splitter/combiners to create 4 digital channels
   which can interface with the :ref:`XUD1A<xud1a>` and
   :adi:`MxFE <resources/evaluation-hardware-and-software/evaluation-boards-kits/eval-ad9081>`.
   The tables below show how these connections should be made:

   .. list-table::
      :header-rows: 1

      - - Connection #
        - Stingray Connectors (SMPM)
        - Splitter # (Ports 1/2) (SMA)
      - - 1
        - J1, J3
        - 1
      - - 2
        - J2, J4
        - 2
      - - 3
        - J6, J8
        - 3
      - - 4
        - J5, J7
        - 4

   .. list-table::
      :header-rows: 1

      - - Connection #
        - XUD1A Connectors (SMA)
        - Splitter # (Sum Port) (SMA)
      - - 1
        - J1D
        - 1
      - - 2
        - J1C
        - 2
      - - 3
        - J1B
        - 3
      - - 4
        - J1A
        - 4

   The connection between the :ref:`XUD1A<xud1a>` and :ref:`Stingray<stingray>`
   boards should look similar to the one in the image below:

   .. image:: ../images/xud1a_stingray_connection.jpg
    :width: 600

#. Insert SD card into the SD card socket on the FPGA
#. Configure :xilinx:`ZCU102` for SD card boot mode (mode SW6[4:1] switch in the
   position **OFF,OFF,OFF,ON** as seen in the below picture)

   .. image:: ../../images/zcu102_1p0_bootmode.jpg
      :width: 400

#. Plug-in an RJ45 Ethernet cable from your router/switch to the Ethernet port
   on the FPGA board

   .. note::

     Modifications to the network settings can be made following the guidance
     detailed on the :dokuwiki:`Network Configuration <resources/tools-software/linux-software/network-config>`
     wiki

   .. _xbdp quickstart zcu102 uart:

#. Connect USB UART J83 (Micro-USB) to your host PC

   .. note::

     If the USB to UART bridge is not installed or automatically recognized,
     then a `driver <https://www.silabs.com/software-and-tools/usb-to-uart-bridge-vcp-drivers?tab=downloads>`_
     must be installed. This will allow control using the UART connection
     through PuTTy or other SSH/Telnet Client

   The COM port can be identified through the device manager as the standard COM
   port:

   .. image:: ../images/comport.png
      :width: 600

#. (Optional) Connect a monitor to the FPGA by HDMI, and a mouse and a keyboard.
   This requires setting the :xilinx:`ZCU102` in USB Host Mode by configuring
   the jumpers as indicated below:

   - Shunt J7
   - J109 → Shunt pins 2-3
   - J110 → Shunt pins 2-3
   - J112 → Shunt pins 1-2
   - J113 → Shunt pins 1-2

   .. image:: ../../images/zcu102_usb_host_mode.jpg
      :width: 400

   .. note::

     If Display Port output is not working (and control using the UART
     connection through PuTTy), try
     :dokuwiki:`this <resources/eval/user-guides/ad-fmcomms2-ebz/software/linux/zynqmp#displayport_-_no_picture>`
     procedure at the bottom of the page

#. Connect the provided power adapters for the :xilinx:`ZCU102`,
   :ref:`XUD1A<xud1a>`, and :ref:`Stingray <stingray>` boards
#. Turn on the power switch on the FPGA board
#. Observe Kernel and serial console output messages on your terminal (use the
   first ttyUSB or COM port registered)

XBDP_SimpleRx.m
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

This script is to be used with the Analog Devices X-Band Platform to demonstrate
relatively simple MATLAB control of the system. It allows the user to configure
the Rx aspects of the system by using the rx = adi.AD9081.Rx and sray =
adi.Stingray system objects.

.. image:: ../images/xbdp_simplerx_sampledomainplot.png
   :width: 800

.. image:: ../images/xbdp_simplerx_sampledomainplot_2.png
   :width: 800

.. image:: ../images/xbdp_simplerx_freqdomainplot_1.png
   :width: 800

.. image:: ../images/xbdp_simplerx_freqdomainplot_2.png
   :width: 800

Using this script as a basis, the user can modify the script for their own use
case such that they can:

-  View Analog Array Channel Mapping Using, For Example, **sray.ArrayMap**
-  Change Rx NCO Frequencies Using, For Example, **rx.MainNCOFrequencies** or
   **rx.ChannelNCOFrequencies**
-  Change Rx NCO Phases Using, For Example, **rx.MainNCOPhases** or
   **rx.ChannelNCOPhases**
-  Change Rx Analog per Channel Gain Using, For Example, **sray.RxGain**
-  Change Rx Analog per Channel Phases Using, For Example, **sray.RxPhase**
-  Change Rx Analog per Channel Attenuation Using, For Example, **sray.RxAttn**
-  Latch Rx Analog Settings Using, For Example, **sray.LatchRxSettings**
-  Capture Simultaneous Complex-Valued Rx Data for All Enabled Channels:
   **data=rx()**
-  Beam steer using **sray.SteerRx(azimuth,elevation,arrayPhaseOffsets)**
-  Beam taper using **sray.TaperRx(window,gain,arrayGainOffsets)**
-  Analyze and Post-Process Captured Waveforms

Matlab Channel Mapping
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

The Matlab attribute mapping is linearly indexed and matches that of the hardware
channel mapping as shown in the figure below.

.. image:: ../images/channel_mapping.png
   :width: 400

.. tip::
   For example, sray.RxGain(1) will modify the gain for channel 1 and sray.RxGain(23)
   modifies the gain for channel 23. sray.RxGain = (127\*ones(2,4) zeros(2,4);zeros(2,4)
   127\*ones(2,4)); will set the gain for subarray 1 and 3 to max gain and subarray 2
   and 4 to 0 gain.

For additional information, please check :ref:`xbdp additional hardware`
and :ref:`xbdp additional software`.
